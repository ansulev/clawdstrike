# Phase 1 — UI Design Contract

> Visual and interaction contract for the Activity Bar + Sidebar Shell phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Manual shadcn-style (no components.json, no CLI) |
| Preset | not applicable |
| Component library | base-ui (via `@base-ui/react`) + cva |
| Icon library | Custom sigil SVGs (`sidebar-icons.tsx`) + `@tabler/icons-react` for utility icons |
| Font | Inter (sans), Syne (display), JetBrains Mono (mono) |

Source: Existing `globals.css` theme tokens, `button.tsx` imports, `sidebar-icons.tsx`.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, sidebar section divider margins |
| sm | 8px | Compact element spacing, activity bar icon internal padding |
| md | 16px | Default element spacing, sidebar panel horizontal padding, panel header horizontal padding |
| lg | 24px | Section padding, activity bar vertical grouping gap |
| xl | 32px | Layout gaps between major zones |
| 2xl | 48px | Activity bar width (fixed) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Activity bar width is exactly 48px (2xl token) per ABAR-01
- Sidebar collapsed width removed -- the activity bar replaces it at 48px
- Sidebar expanded width: 240px (increased from current 200px to accommodate panel content in Phase 2; Phase 1 sets this as default in store)
- Sidebar minimum width before collapse threshold: 120px
- Titlebar height: 36px (existing, unchanged)
- StatusBar height: 24px (existing, unchanged)

## Typography

This phase introduces no new typographic elements. All text in Phase 1 is either existing (titlebar, statusbar) or uses the established type scale.

Weights: 400 (regular) + 600 (semibold). Two weights only.

| Role | Size | Weight | Line Height | Font | Usage in Phase 1 |
|------|------|--------|-------------|------|-------------------|
| Caption | 10px | 600 (semibold) | 1.3 | JetBrains Mono | Sidebar panel header section labels |
| Label | 11px | 400 (regular) | 1.3 | JetBrains Mono | Activity bar icon tooltips |
| Body | 14px | 400 (regular) | 1.5 | Inter | Sidebar panel placeholder text |
| Heading-sm | 14px | 600 (semibold) | 1.2 | Syne | Sidebar panel title |

Deferred: Micro (9px) is used only for tooltip micro-text (posture counts) shipping in Phase 2+. It will be declared in that phase's UI-SPEC.

Source: Existing type scale documented in `globals.css` lines 76-86.

In Phase 1, the sidebar panel container renders a **placeholder** for each panel. Actual panel content (lists, trees, status views) ships in Phase 2.

Each sidebar panel has a consistent header:

```
+-------------------------------------+
| [Panel Title]            [... menu] |  14px Syne semibold, #ece7dc
```

- Panel title: 14px Syne semibold, color `#ece7dc`
- Header height: 32px
- Header bottom border: 1px solid `#2d3240` at 40% opacity
- Menu button: ghost icon button, hidden until Phase 2+
- Horizontal padding: 16px (md token)

## State Management Specification

### activity-bar-store

```typescript
interface ActivityBarState {
  activeItem: ActivityBarItemId;
  sidebarVisible: boolean;
  sidebarWidth: number;

  actions: {
    setActiveItem: (id: ActivityBarItemId) => void;
    toggleItem: (id: ActivityBarItemId) => void;   // click handler: sets active or collapses
    toggleSidebar: () => void;                     // Cmd+B
    showPanel: (id: ActivityBarItemId) => void;    // show sidebar + set active
    setSidebarWidth: (width: number) => void;
    collapseSidebar: () => void;
  };
}

type ActivityBarItemId =
  | "heartbeat"
  | "sentinels"
  | "findings"
  | "explorer"
  | "library"
  | "fleet"
  | "compliance";
// "settings" is NOT an item ID -- it opens as an editor tab
```

Default state:
- `activeItem`: `"explorer"`
- `sidebarVisible`: `true`
- `sidebarWidth`: `240`

Store location: `src/features/activity-bar/stores/activity-bar-store.ts`
Pattern: `createSelectors` + `immer` middleware, matching existing store conventions.

## Commands Specification

### New Commands (Phase 1)

| Command ID | Title | Category | Keybinding | Implementation |
|------------|-------|----------|------------|----------------|
| `sidebar.toggle` | Toggle Sidebar | View | `Cmd+B` | `activityBarStore.getState().actions.toggleSidebar()` |
| `sidebar.explorer` | Show Explorer | Sidebar | `Cmd+Shift+E` | `activityBarStore.getState().actions.showPanel("explorer")` |

### Modified Commands (Phase 1)

The existing sidebar collapse toggle (currently dispatching `SET_SIDEBAR_COLLAPSED` to multi-policy-store) must be rewired to use `activity-bar-store.toggleSidebar()` instead.

### Settings Click Behavior

Clicking the Settings activity bar icon does NOT change `activeItem`. It navigates to `/settings` (or in Phase 2+, opens settings as an editor tab via `openApp`). The settings icon shows active state based on current route, not `activeItem`.
