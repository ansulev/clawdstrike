## Design System

| Property | Value |

## Spacing Scale

Declared values (must be multiples of 4) -- inherited from Phase 1:

| Token | Value | Usage in Phase 2 |
|-------|-------|-------------------|
| xs | 4px | Icon gaps, status dot margin, inline padding within list items |
| sm | 8px | List item vertical padding, compact element gaps, filter input padding |
| md | 16px | Panel body horizontal padding, section gaps within panels |
| lg | 24px | Section dividers between panel zones (e.g. findings vs intel sections) |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Activity bar width (unchanged from Phase 1) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- PaneTabBar height: 36px (matches titlebar height for visual consistency)
- Tab close button hit target: 20x20px (minimum touch target for close action within constrained tab width)
- Sidebar panel list item height: 32px (compact list density matching VS Code explorer items)
- Posture ring size in HeartbeatPanel: 80px diameter (compact sidebar-width adaptation of 200px home page ring)
- Stat count block height: 44px (accommodates 9px label + 20px value + padding)

### Shared Panel Structure

Every sidebar panel follows this consistent layout:

```
+---------------------------------------------+
| [SECTION LABEL]              [action icons]  |  10px JetBrains Mono semibold, #6f7f9a, uppercase tracking-wider
+---------------------------------------------+
| [search/filter input]                        |  11px JetBrains Mono, #ece7dc
+---------------------------------------------+
| [scrollable list content]                    |
|                                              |
|   [ item ]                                   |  32px tall list rows
|   [ item ]                                   |
|   [ item ]                                   |
|                                              |
+---------------------------------------------+
| [footer status]                              |  9px JetBrains Mono, #6f7f9a/40
+---------------------------------------------+
```

Panel header: reuse existing Phase 1 panel header (32px, Syne semibold 14px, `#ece7dc`, bottom border `#2d3240` at 40%).

Section labels within panels: 10px JetBrains Mono semibold, `#6f7f9a`, uppercase, `letter-spacing: 0.08em`, 12px horizontal padding, 8px vertical padding.

Toolbar icon buttons: 12px icon size, stroke 1.5, `p-1`, `#6f7f9a/60` default, `#ece7dc` on hover, `#131721/40` background on hover. Same pattern as ExplorerPanel toolbar.

Filter inputs: 11px JetBrains Mono, `#ece7dc` text, `#0b0d13` background, `#2d3240` border, `#d4a84b/40` focus border, 12px left icon, `pl-7 pr-2 py-1`.

## PaneTabBar Specification (PANE-03)

### Layout

The PaneTabBar replaces the current single-label header in `PaneContainer`. It renders all views in a `PaneGroup` as horizontal tabs.

```
+--------------------------------------------------------------------+
| [Home] [x] | [Editor] [x] | [Sentinels] [x] |    [split] [split] |
+====================================================================+
| (active tab underline indicator, 2px gold)                         |
+--------------------------------------------------------------------+
```

### Tab Structure

Each tab:
- Height: 36px
- Horizontal padding: 12px (includes close button space)
- Max width: 160px (text truncates with ellipsis)
- Min width: 80px
- Content: label text (11px JetBrains Mono weight 500, tracking 0.04em) + close button (14px `IconX`, shown only on hover or when tab is active)
- Gap between label and close: 4px

### Tab Bar Container

- Height: 36px
- Background: `#0b0d13` (surface)
- Bottom border: 1px solid `#202531`
- Overflow: horizontal scroll (hide scrollbar via CSS), no wrapping
- Right side: split buttons (vertical + horizontal split icons), matching existing pane header style
- Close-pane button: only shown when `paneCount > 1` (existing behavior)

### Active Tab Indicator

The active tab has a 2px-tall `#d4a84b` underline at its bottom edge, matching the activity bar's left-edge indicator metaphor rotated 90 degrees:

```css
position: absolute;
bottom: 0;
left: 8px;
right: 8px;
height: 2px;
border-radius: 9999px 9999px 0 0;
background-color: #d4a84b;
```

### Close Tab Behavior

- Close button visibility: hidden by default, visible on tab hover or when tab is active
- Close button hover: background `#2a1115`, icon color `#ffb8b8` (existing destructive hover pattern)
- Closing the last tab in a pane: if pane has siblings, close the entire pane (existing behavior). If it's the only pane, open Home tab as fallback
- Middle-click on tab: close that tab (VS Code convention)

### Tab Drag (NOT in Phase 2)

Tab reordering via drag is deferred to v2 requirement ALYT-01. Tabs render in insertion order.

## Pane Store Enhancement Specification (PANE-01)

### New Actions

```typescript
interface PaneStore {
  // ... existing actions unchanged ...

  /** Open a route as a new tab in the active pane, or focus an existing tab with that route. */
  openApp: (route: string, label?: string) => void;

  /** Close a specific view (tab) within a pane group. */
  closeView: (paneId: string, viewId: string) => void;

  /** Switch to a specific view (tab) within a pane group. */
  setActiveView: (paneId: string, viewId: string) => void;
}
```

### openApp Behavior

1. Normalize the route via `normalizeWorkbenchRoute(route)`.
2. Search ALL pane groups for an existing view with the same normalized route.
3. If found: set that pane as active (`setActivePane`), set that view as active within the pane (`activeViewId`).
4. If not found: add a new `PaneView` to the active pane group's `views[]` array, set it as `activeViewId`.
5. Label falls back to `getWorkbenchRouteLabel(route)` if not provided.

### closeView Behavior

1. Remove the view from the pane group's `views[]`.
2. If the closed view was `activeViewId`, switch to the adjacent view (prefer right neighbor, fall back to left).
3. If the pane group has zero views remaining: close the pane entirely (using existing `closePane` logic).

### Default State

On app launch: single pane group with one view: `{ route: "/home", label: "Home" }`. This is the existing behavior.

## Accessibility

### Sidebar Panels

| Element | Requirement |
|---------|-------------|
| Panel list (`SentinelPanel`, `FindingsPanel`, etc.) | `role="listbox"`, `aria-label="{Panel Name}"` |
| List item | `role="option"`, `aria-selected={isActive}`, `tabindex` managed for roving focus |
| Collapsible section header | `role="button"`, `aria-expanded={isOpen}`, `aria-controls` pointing to section content |
| Filter input | `aria-label="Filter {panel name}"`, visible label not required (search icon is decorative) |
| Status dots | `aria-hidden="true"` (status communicated via text label) |
| Severity badges | Text content is accessible (CRIT, HIGH, etc.) |
| Quick links in HeartbeatPanel | Standard `<button>` elements, not `<a>` tags (they trigger paneStore actions, not navigation) |

### PaneTabBar

| Element | Requirement |
|---------|-------------|
| Tab bar container | `role="tablist"`, `aria-label="Open editors"`, `aria-orientation="horizontal"` |
| Individual tab | `role="tab"`, `aria-selected={isActive}`, `aria-controls="pane-content-{paneId}"`, keyboard focus via Arrow Left/Right |
| Tab close button | `aria-label="Close {tab label}"`, `tabindex="-1"` (not in tab order; accessible via mouse/keyboard shortcut) |
| Pane content area | `role="tabpanel"`, `id="pane-content-{paneId}"`, `aria-labelledby` pointing to active tab |
| Keyboard: Arrow Left/Right | Moves focus between tabs within the tab bar |
| Keyboard: Enter/Space on tab | Activates the focused tab |
| Keyboard: Cmd+W | Closes the active tab (new keyboard shortcut for Phase 2) |

### Focus Management

When a sidebar panel item is clicked and opens an editor tab:
1. Focus moves to the newly opened editor tab content area
2. The sidebar panel retains its scroll position
3. The sidebar remains visible (no automatic collapse)
