## Right Sidebar Specification (RBAR-01, RBAR-02, RBAR-03)

### Layout

The right sidebar mirrors the left sidebar's structural pattern but is simpler: it does not have an activity bar rail. It is a single-panel container.

```
+---------------------------------------------------+
| [Panel Title: "Speakeasy"]    [close/collapse btn] |  32px header
+---------------------------------------------------+
| (SpeakeasyPanel content, scrollable)               |
|                                                     |
|   [RoomHeader]                                      |
|   [MessageList]                                     |
|   [ComposeArea / SentinelRequestForm]               |
|   [ClassificationFooter]                            |
|                                                     |
+---------------------------------------------------+
```

### Panel Header

Matches left sidebar panel header pattern exactly:

- Height: 32px
- Title: 14px Syne semibold, `#ece7dc`
- Bottom border: 1px solid `#2d3240` at 40% opacity
- Horizontal padding: 16px (md token)
- Right side: collapse button (14px `IconChevronsRight`, `#6f7f9a`, hover `#ece7dc`)

### SpeakeasyPanel Integration

The existing `SpeakeasyPanel` component currently renders as a fixed overlay with backdrop (`fixed top-0 right-0 bottom-0 z-50 w-96`). Phase 3 removes the overlay/backdrop pattern and renders the SpeakeasyPanel as inline content within the `RightSidebar` container.

Changes to SpeakeasyPanel rendering:
1. Remove the `fixed` positioning and `z-50` stacking context
2. Remove the backdrop `div` (no more click-outside-to-close via overlay)
3. The panel becomes a flex-column child of `RightSidebar`, using `flex-1 min-h-0` to fill available space
4. Width is determined by the RightSidebar container (no longer hardcoded `w-96`)
5. The `onClose` prop is wired to `rightSidebarStore.actions.hide()`
6. The `isOpen` prop is derived from `rightSidebarStore.visible`

The SpeakeasyPanel's internal structure (RoomHeader, MessageList, compose area, ClassificationFooter) remains completely unchanged. Only the outer shell changes from overlay to inline.

### Resize Behavior

| Property | Value | Source |
|----------|-------|--------|
| Default width | 320px | right-sidebar-store default |
| Min width | 200px | collapse threshold |
| Max width | 480px | maximum resize limit |
| Resize handle width | 4px | mirrors left sidebar |
| Collapse threshold | Below 200px while dragging | triggers hide |

## AuditTailPanel Specification (BPAN-01, BPAN-02)

### Purpose

A lightweight, streaming audit log panel for the bottom pane's 4th tab. Shows the last N audit entries with auto-refresh. This is NOT the full AuditLog page (which remains openable as an editor tab). This is a compact tail view.

### Layout

```
+----------------------------------------------------------------------+
| [source badge] [decision/event badge] summary text        timestamp  |  28px row
| [source badge] [decision/event badge] summary text        timestamp  |
| [source badge] [decision/event badge] summary text        timestamp  |
| [source badge] [decision/event badge] summary text        timestamp  |
|                                                                      |
|                        (scrollable, newest at top)                    |
+----------------------------------------------------------------------+
| 47 events -- auto-refresh: on     [clear] [pause/resume] [open full] |  footer
+----------------------------------------------------------------------+
```

### Data Source

Uses the same `useLocalAudit()` hook from `@/lib/workbench/local-audit.ts` that the full AuditLog page uses. Shows only local events (no fleet fetch -- fleet data requires the full AuditLog editor tab).

Display limit: Show last 50 events (sliced from the `events` array which is already sorted newest-first).

### Auto-Refresh

The `useLocalAudit()` hook uses `useSyncExternalStore` which automatically re-renders when `emitAuditEvent()` is called anywhere in the app. This provides built-in auto-refresh without polling. No interval timer needed.

A `paused` state boolean in the component (local `useState`, not in the store) freezes the displayed snapshot. When paused, new events continue to accumulate in the store but the displayed list is frozen. Unpausing snaps to the latest state.

### Entry Row Structure

Each 28px row:
- Left: source badge (9px uppercase, colored pill matching `SOURCE_COLORS` from audit-log.tsx)
- Center-left: decision/event-type badge (9px uppercase, colored pill using `DECISION_COLORS` or `LOCAL_EVENT_TYPE_COLORS` from audit-log.tsx)
- Center: summary text (10px JetBrains Mono, `#ece7dc/60`, truncated with ellipsis)
- Right: relative timestamp (9px JetBrains Mono, `#6f7f9a/40`, e.g. "2m ago", "just now")

Row hover: background `#131721` at 40% opacity.
Row click: expands inline to show `details` JSON (same `<pre>` pattern as audit-log.tsx detail rows, but simplified to 3-line max-height in the compact panel).

### Footer Bar

- Height: 28px
- Background: `#07090f` (matches bottom pane background)
- Border-top: 1px solid `#202531`
- Left: event count text (10px, `#6f7f9a/60`, e.g. "47 events")
- Left: auto-refresh status (10px, `#6f7f9a/40`, "auto-refresh: on" or "paused")
- Right: action buttons (ghost style, 12px icons, `#6f7f9a/60` default, `#ece7dc` hover)
  - Clear button: `IconTrash`, clears local audit events
  - Pause/Resume button: `IconPlayerPause` / `IconPlayerPlay`, toggles pause state
  - Open full button: `IconExternalLink`, calls `paneStore.openApp("/audit", "Audit Log")`

### Empty State

When no local events exist:

```
+----------------------------------------------------------------------+
|                                                                      |
|            [IconFileAnalytics, 24px, #6f7f9a/30]                     |
|        No audit events recorded yet.                                 |
|   Events appear here as you use the workbench.                       |
|                                                                      |
+----------------------------------------------------------------------+
```

Centered vertically. Icon: 24px `IconFileAnalytics`. Text: 11px JetBrains Mono, `#6f7f9a/40`. Matches existing bottom pane empty patterns.

```
DesktopLayout (flex-col, h-screen, w-screen)
  Titlebar                        (shrink-0, h-[36px])
  CrashRecoveryBanner             (conditional, shrink-0)
  InitCommands + ShortcutProvider + CommandPalette  (invisible)
  MainArea (flex-row, flex-1, min-h-0)
```

Key layout decisions:
- The right sidebar sits OUTSIDE the `<main>` flex column (editor area + bottom pane). This means the right sidebar spans the full height of the main area, not just the editor portion.
- The right sidebar resize handle sits between the `<main>` element and the `RightSidebar`.
- When the right sidebar is hidden (`visible: false`), both `RightSidebarResizeHandle` and `RightSidebar` are not rendered (conditional rendering, not `display: none`).
- The bottom pane is bounded horizontally by the editor area only -- it does NOT extend under the right sidebar. This matches VS Code behavior.
