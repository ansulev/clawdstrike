Define these types (using interfaces, not classes):

1. **PluginManifest** (main interface):
   - `id: string` -- reverse-domain ID, e.g. "clawdstrike.egress-guard"
   - `name: string` -- human-readable name
   - `displayName: string` -- display name for UI
   - `description: string`
   - `version: string` -- semver
   - `publisher: string`
   - `categories: PluginCategory[]`
   - `trust: PluginTrustTier` -- "internal" | "community" | "mcp"
   - `activationEvents: ActivationEvent[]`
   - `main?: string` -- entry point path for plugin code
   - `contributions?: PluginContributions` -- all contribution point declarations
   - `installation?: InstallationMetadata` -- distribution metadata

2. **PluginTrustTier** -- type alias: `"internal" | "community" | "mcp"`

3. **PluginCategory** -- type alias: `string` (open, like Phase 1 pattern). Add `const BUILTIN_PLUGIN_CATEGORIES = ["guards", "detection", "intel", "compliance", "ui", "integration"] as const`

4. **ActivationEvent** -- type alias: `string`. Document known patterns in JSDoc: `"onStartup"`, `"onFileType:{type}"`, `"onCommand:{id}"`, `"onGuardEvaluate:{id}"`

5. **PluginContributions** (object grouping all contribution types):
   - `guards?: GuardContribution[]`
   - `commands?: CommandContribution[]`
   - `keybindings?: KeybindingContribution[]`
   - `fileTypes?: FileTypeContribution[]`
   - `detectionAdapters?: DetectionAdapterContribution[]`
   - `activityBarItems?: ActivityBarItemContribution[]`
   - `editorTabs?: EditorTabContribution[]`
   - `bottomPanelTabs?: BottomPanelTabContribution[]`
   - `rightSidebarPanels?: RightSidebarPanelContribution[]`
   - `statusBarItems?: StatusBarItemContribution[]`
   - `threatIntelSources?: ThreatIntelSourceContribution[]`
   - `complianceFrameworks?: ComplianceFrameworkContribution[]`

6. **GuardContribution**:
   - `id: string` -- guard ID to register in guard-registry
   - `name: string` -- display name
   - `technicalName: string`
   - `description: string`

7. **CommandContribution**:

   - `shortcut?: string`
   - `when?: string` -- contextual visibility expression (for future use)

8. **KeybindingContribution**:

   - `when?: string`

9. **FileTypeContribution**:
   - `id: string`

11. **ActivityBarItemContribution**:

18. **InstallationMetadata**:
    - `downloadUrl: string`
    - `size: number` -- bytes

    - `minWorkbenchVersion?: string` -- semver compatibility range

3. Export `validateManifest(input: unknown): ManifestValidationResult`:
   - Do NOT use Zod or io-ts (per Claude's Discretion from CONTEXT.md, use manual type guards -- lighter weight, zero deps)
   - Check input is a non-null object
   - Validate required string fields: id, name, displayName, description, version, publisher
   - Validate `version` matches basic semver pattern: `/^\d+\.\d+\.\d+/` (allow pre-release suffixes)
   - Validate `trust` is one of "internal", "community", "mcp"
   - Validate `activationEvents` is an array of strings (if present)
   - Validate `categories` is an array of strings (if present)
   - Validate `contributions.guards[]`: each must have id (string), name (string), category (string), configFields (array)
   - Validate `contributions.commands[]`: each must have id (string), title (string)
   - Validate `contributions.fileTypes[]`: each must have id (string), label (string), extensions (string[])
   - Validate `installation` if present: downloadUrl (string), size (number > 0), checksum (string, 64 hex chars for SHA-256), signature (string, non-empty)
   - Accumulate ALL errors (do not short-circuit on first error)
   - Return `{ valid: errors.length === 0, errors }`
