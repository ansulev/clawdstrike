    **Part A: Create plugins-browser.tsx**

    Create PluginsBrowser component that shows two sections: "Installed Plugins" and "Available Plugins".

    State:
    - `searchQuery: string` (useState, default "")
    - `installedPlugins: RegisteredPlugin[]` (useState, populated from pluginRegistry.getAll())
    - `availablePlugins: RegistrySearchResult[]` (useState, populated from registryClient)
    - `loading: boolean` (useState, for registry fetch)

    On mount (useEffect):
    - Read installed plugins from `pluginRegistry.getAll()`
    - Subscribe to pluginRegistry events ("registered", "unregistered", "stateChanged") to refresh the installed list on changes. Return the unsubscribe disposable from the effect cleanup.
    - Fetch popular plugins from `registryClient.getPopular(20)` for the "Available" section. Wrap in try/catch -- on failure, show empty state with "Registry unavailable" message.

    On search:
    - If searchQuery is non-empty, call `registryClient.search(searchQuery)` debounced (300ms via setTimeout). Show results in the Available section.
    - Also filter installed plugins client-side by name/description match.
    - If searchQuery is empty, show the popular plugins list.

    Layout:
    - Search input at top: `<input>` with IconSearch, styled like catalog-browser search (rounded-lg bg-[#131721]/50 border border-[#2d3240]/40 text-sm text-[#ece7dc] placeholder-[#6f7f9a]/50 pl-9 pr-3 py-2)
    - "Installed Plugins" section with green dot indicator (same pattern as library-gallery sections)
      - Grid: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4`
      - Each installed plugin rendered as `<PluginCard manifest={p.manifest} state={p.state} error={p.error} onUninstall={...} />`
      - Empty state when no plugins installed: dashed border card with "No plugins installed yet" message
    - "Available Plugins" section with gold dot indicator
      - Same grid layout
      - Each available plugin rendered as `<PluginCard manifest={...} state="not-installed" onInstall={...} />`
      - For registry results that are already installed (by matching name), skip them or show them in installed section only.
      - Loading state: show "Loading..." with IconLoader2 animate-spin
      - Empty state when registry unavailable: "Connect to a registry to browse available plugins"

    const asManifest = (r: RegistrySearchResult): PluginManifest => ({
