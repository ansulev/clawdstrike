    GREEN: Create plugin-installer.ts with:

    ```typescript
    export interface InstallOptions {
      registry?: PluginRegistry;
      loader?: PluginLoader;
    }
    ```

    **installPlugin(manifest: PluginManifest, options?: InstallOptions): Promise<void>**
    Orchestrates the full install lifecycle:
    1. Get registry (options?.registry ?? pluginRegistry singleton)
    2. Get loader (options?.loader ?? pluginLoader singleton)
    3. Call registry.register(manifest) -- this validates and sets state to "installed"
    4. Call loader.loadPlugin(manifest.id) -- this runs trust check, routes contributions, calls activate()
    5. On error during loadPlugin, the loader itself sets state to "error", so no additional cleanup needed here -- the plugin stays registered with error state so the user can see what went wrong.

    **uninstallPlugin(pluginId: string, options?: InstallOptions): Promise<void>**
    Orchestrates the full uninstall lifecycle:
    1. Get registry and loader from options or singletons
    2. Check if plugin exists in registry -- if not, return early (no-op)
    3. Call loader.deactivatePlugin(pluginId) -- this cleans up contributions and sets state to "deactivated"
    4. Call registry.unregister(pluginId) -- this removes the plugin entry and emits "unregistered" event

    Export both functions. Do NOT export a class -- these are standalone orchestration functions that compose the registry and loader.

    Import pluginRegistry from "./plugin-registry".
    Import pluginLoader from "./plugin-loader".
  </action>
