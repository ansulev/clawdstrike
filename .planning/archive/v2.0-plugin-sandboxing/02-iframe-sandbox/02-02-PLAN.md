       }

    4. In `loadPlugin()`, after the trust gate (step 1) and before "Set state to activating" (step 2), add the trust tier check:

       ```typescript
       if (manifest.trust === "community") {

       }
       // ... existing internal plugin path continues below ...
       ```

    5. Add private method `loadCommunityPlugin(pluginId: string, manifest: PluginManifest, disposables: Disposable[])`:
       a. Set state to "activating"
       b. Resolve plugin code: `const pluginCode = this.resolvePluginCode ? await this.resolvePluginCode(manifest) : "";`
       c. Create iframe element: `const iframe = document.createElement("iframe")`
       d. Set `iframe.sandbox.add("allow-scripts")` -- explicitly do NOT add allow-same-origin
       e. Set `iframe.srcdoc = buildPluginSrcdoc({ pluginCode, pluginId })`
       f. Set iframe style: `width: 0; height: 0; border: none; position: absolute; visibility: hidden` (plugin iframes are invisible by default -- UI contribution points will manage visible rendering later)
       g. Append iframe to container: `(this.iframeContainer ?? document.body).appendChild(iframe)`
       h. Wait for iframe load: `await new Promise(resolve => { iframe.onload = resolve })`
       i. Get contentWindow: `const targetWindow = iframe.contentWindow!`
       j. Create bridge host: `const host = new PluginBridgeHost({ pluginId, targetWindow })`
       k. Create message handler: `const messageHandler = (e: MessageEvent) => host.handleMessage(e)`
       l. Attach: `window.addEventListener("message", messageHandler)`
       m. Route manifest contributions if present (same as existing routeContributions but through the bridge host -- call `this.routeContributions(manifest, disposables)` since the bridge host handles the actual plugin->host RPC, while manifest.contributions routes static declarations)
       n. Store: `this.loadedPlugins.set(pluginId, { disposables, module: null, bridgeHost: host, messageHandler, iframe })`
       o. Set state to "activated"

    6. Update `deactivatePlugin()` to handle community plugin cleanup:
       After existing `loaded.module.deactivate?.()` call, add:
       ```typescript
       // Clean up community plugin resources
       if (loaded.bridgeHost) {
         loaded.bridgeHost.destroy();
       }
       if (loaded.messageHandler) {
         window.removeEventListener("message", loaded.messageHandler);
       }
       if (loaded.iframe) {
         loaded.iframe.remove();
       }
       ```
       Guard the module.deactivate call: `if (loaded.module?.deactivate) { loaded.module.deactivate(); }`

    7. Wrap the community loading path in try/catch just like the existing internal path -- on error, clean up iframe, set state to "error".
