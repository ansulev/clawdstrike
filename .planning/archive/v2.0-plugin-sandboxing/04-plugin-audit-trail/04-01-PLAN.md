    **receipt-middleware.ts:**
    Create ReceiptMiddlewareOptions interface:
    - pluginId: string
    - pluginVersion: string
    - publisher: string
    - trustTier: PluginTrustTier
    - secretKeyHex: string | null (null = unsigned/dev mode)
    - publicKeyHex: string (for signer_public_key field)
    - store: PluginReceiptStore (injected for testability)

    Create createReceiptMiddleware(options: ReceiptMiddlewareOptions) returning:
    - recordAllowed(method: string, params: unknown, durationMs: number): Promise<void>
      1. Calls createReceiptContent with result "allowed"
      2. If secretKeyHex is provided, calls signCanonical(content, secretKeyHex) to get signature
      3. If secretKeyHex is null, signature = ""
      4. Stores PluginActionReceipt { content, signature, signer_public_key } via store.add()
    - recordDenied(method: string, params: unknown, permissionChecked: string): Promise<void>
      1. Calls createReceiptContent with result "denied", durationMs = 0
      2. Signs and stores -- ALWAYS, no verbosity check (AUDIT-02)
    - recordError(method: string, params: unknown, durationMs: number): Promise<void>
      1. Same as recordAllowed but with result "error"

    Import signCanonical from operator-crypto.ts. Import createReceiptContent from receipt-types.ts.

    **bridge-host.ts modifications:**
    Add optional `receiptMiddleware` field to BridgeHostOptions:
    - receiptMiddleware?: { recordAllowed: ..., recordDenied: ..., recordError: ... } | null

    Store as private field. In handleMessage():
    1. After permission denial (sendError with PERMISSION_DENIED), call this.receiptMiddleware?.recordDenied(method, params, requiredPerm). Use void operator to not await (fire-and-forget, non-blocking).
    2. After successful handler dispatch (sendResponse), call this.receiptMiddleware?.recordAllowed(method, params, durationMs). Compute durationMs by capturing performance.now() before handler call and computing delta.
    3. After handler error (sendError with INTERNAL_ERROR), call this.receiptMiddleware?.recordError(method, params, durationMs).

    Receipt generation is fire-and-forget (void the promise) so it does not block bridge response time. Errors in receipt generation are caught and logged to console.warn (fail-open for audit, fail-closed for security).
