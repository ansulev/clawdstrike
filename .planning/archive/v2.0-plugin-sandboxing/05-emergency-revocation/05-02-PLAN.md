<objective>
Wire the revocation infrastructure to hushd SSE for fleet-wide coordination: an SSE listener that handles plugin_revoked events, offline-to-online reconnection sync, and a UI badge component showing revocation status.

Purpose: This completes the emergency revocation capability. When an operator calls POST /api/v1/plugins/{plugin_id}/revoke on hushd, every connected workbench instance kills the plugin automatically. Offline instances catch up on reconnect.

Output: PluginRevocationClient (SSE listener + reconnect sync), PluginRevocationBadge component, updated PluginLoader for reconnect awareness.
</objective>
