  <behavior>
    - registerView({id: "p.v", slot: "editorTab", label: "Test", component: TestComponent}) stores the registration and getView("p.v") returns it
    - getViewsBySlot("editorTab") returns only views with slot === "editorTab", sorted by priority ascending
    - getViewsBySlot("bottomPanelTab") returns empty array when no views registered for that slot
    - registerView() returns a dispose function; calling it removes the view from the map and getView returns undefined
    - registerView() with a duplicate id throws Error('View "p.v" already registered')
    - onViewRegistryChange(listener) causes listener to be called on register and on dispose
    - onViewRegistryChange returns an unsubscribe function; after calling it, listener is not called
    - getViewsBySlot returns a stable snapshot reference when no changes occur (same object identity)
  </behavior>
  <action>
    Create `apps/workbench/src/lib/plugins/view-registry.ts` following the exact Map + snapshot + listeners architecture from status-bar-registry.ts:

    1. Define and export types:
       - `ViewSlot` union type: `"editorTab" | "activityBarPanel" | "bottomPanelTab" | "rightSidebarPanel" | "statusBarWidget" | "gutterDecoration" | "contextMenuItem"`
       - `ViewProps` interface: `{ viewId: string; isActive: boolean; storage: { get(key: string): unknown; set(key: string, value: unknown): void } }`
       - `ViewRegistration` interface: `{ id: string; slot: ViewSlot; label: string; icon?: string; component: ComponentType<any>; priority?: number; meta?: Record<string, unknown> }`

    2. Module-level state (NOT a class, matching status-bar-registry pattern):
       - `const viewMap = new Map<string, ViewRegistration>()`
       - `const listeners = new Set<() => void>()`
       - `let snapshotBySlot = new Map<ViewSlot, ViewRegistration[]>()` (snapshot cache)

    3. Internal helpers:
       - `rebuildSnapshots()`: iterate viewMap, group by slot, sort each group by `(a.priority ?? 100) - (b.priority ?? 100)`, store in snapshotBySlot
       - `notify()`: call rebuildSnapshots(), then call each listener

    4. Exported functions:
       - `registerView(reg: ViewRegistration): () => void` — throws if id exists, sets in map, calls notify, returns dispose function that deletes and notifies
       - `getView(id: string): ViewRegistration | undefined` — direct map lookup
       - `getViewsBySlot(slot: ViewSlot): ViewRegistration[]` — returns snapshot or empty array
       - `onViewRegistryChange(listener: () => void): () => void` — adds to set, returns unsubscribe
       - `useViewsBySlot(slot: ViewSlot): ViewRegistration[]` — wraps useSyncExternalStore(onViewRegistryChange, () => getViewsBySlot(slot))

    5. Convenience export: `export const viewRegistry = { register: registerView, get: getView, getBySlot: getViewsBySlot, onChange: onViewRegistryChange }`

    Write tests FIRST in `__tests__/view-registry.test.ts` matching the status-bar-registry test structure. Use vitest, track disposers in afterEach, test all behaviors listed above. Do NOT test the React hook (useViewsBySlot) in this unit test file — it requires React rendering context.

  <behavior>
    - ViewContainer renders the registration's component, passing viewId, isActive, and storage props
    - When the wrapped component throws during render, ErrorBoundary catches it and renders a fallback with text "Plugin view crashed", the error message, and a "Reload View" button
    - Clicking "Reload View" remounts the component (resets error boundary state)
    - When the component is a React.lazy component that hasn't loaded yet, Suspense renders a loading fallback
    - ViewContainer accepts a slotType prop; when slotType is "editorTab" the Suspense fallback is a full-panel centered spinner; when slotType is "statusBarWidget" it is an inline skeleton
  </behavior>
  <action>
    Create `apps/workbench/src/components/plugins/view-container.tsx`:

    1. `ViewErrorFallback` component (internal):
       - Props: `{ viewId: string; error: Error; resetError: () => void }`
       - Renders a centered vertical layout with:
         - Red text "Plugin view crashed" (`text-[#c45c5c]`)
         - Gray error message (`text-[#6f7f9a]`, max-w-md, truncated)
         - Amber "Reload View" button (`text-[#d4a84b]`, hover:underline, calls resetError onClick)
       - Use the workbench's existing color tokens from status-bar.tsx

    2. `ViewErrorBoundary` class component:
       - Standard React error boundary (getDerivedStateFromError + componentDidCatch)
       - State: `{ hasError: boolean; error: Error | null }`
       - Props: `{ viewId: string; children: ReactNode }`
       - When error: render `<ViewErrorFallback viewId={viewId} error={error} resetError={() => this.setState({ hasError: false, error: null })} />`
       - When no error: render children
       - Use a `key` prop pattern: expose `resetKey` state that increments on reset, use it as key on the children wrapper to force remount

    3. `ViewLoadingFallback` component (internal):
       - Props: `{ slotType: ViewSlot }`
       - For "editorTab" | "activityBarPanel": full height centered spinner with "Loading plugin view..." text
       - For "statusBarWidget": inline small spinner (10px) with no text
       - For all others ("bottomPanelTab", "rightSidebarPanel", etc.): medium spinner with "Loading..." text
       - Use Tailwind classes and a simple CSS spinner (animate-spin on a border-based element), no external spinner library

    4. `ViewContainer` exported component:
       - Props: `{ registration: ViewRegistration; isActive?: boolean; slotType?: ViewSlot; storage?: { get(key: string): unknown; set(key: string, value: unknown): void } }`
       - Default isActive to true, default slotType to registration.slot
       - Default storage to a no-op implementation: `{ get: () => undefined, set: () => {} }`
       - Renders:
         ```
         <ViewErrorBoundary viewId={registration.id}>
           <Suspense fallback={<ViewLoadingFallback slotType={slotType} />}>
             <registration.component viewId={registration.id} isActive={isActive} storage={storage} />
           </Suspense>
         </ViewErrorBoundary>
         ```

    Write tests in `__tests__/view-container.test.tsx`:
    - Test that ViewContainer renders the component with correct props (viewId, isActive, storage)
    - Test that a throwing component triggers the error fallback (renders "Plugin view crashed" and error message text)
    - Test that clicking "Reload View" resets the error boundary (component re-renders)
    - Use vitest + @testing-library/react. Create a simple test component and a deliberately-crashing test component.
    - Suppress console.error during error boundary tests with vi.spyOn(console, 'error').mockImplementation(() => {})
