The simplest extension is: when the secondary pane selector offers tabs, also offer open plugin view tabs. When a plugin view is selected as the secondary pane content, render a ViewContainer for it instead of the SecondaryPanePreview.

1. Import `usePluginViewTabs` from `@/lib/plugins/plugin-view-tab-store`
2. Import `getView` from `@/lib/plugins/view-registry`
3. Import `ViewContainer` from `@/components/plugins/view-container`

4. In `PaneTabSelector`, extend the available options to include open plugin view tabs:
   - After the existing policy tabs, add plugin view tabs as selectable items
   - Plugin tab IDs should be prefixed with `plugin:` to distinguish from policy tab IDs (e.g., `plugin:myPlugin.myView`)
   - Show plugin tabs with a `[Plugin]` label prefix or a distinctive icon color

5. In `SplitEditor`, where the secondary pane renders:
   - Check if `splitTabId` starts with `plugin:`. If yes, extract the viewId = splitTabId.slice(7), look up the ViewRegistration via `getView(viewId)`, and render `<ViewContainer registration={reg} isActive={true} slotType="editorTab" />` instead of `<SecondaryPanePreview>`.

6. When `SET_SPLIT_TAB` is dispatched with a plugin: prefixed ID, the multi-policy-store passes it through as a string (it's already typed as `string | null`). No changes needed to multi-policy-store itself.

7. The existing `PaneTabSelector` uses a `<Select>` dropdown. Add plugin view tabs as additional `<SelectItem>` entries with the `plugin:` prefix.
