**1. Create `gutter-extension-registry.ts`:**

Follow the Map + snapshot + listeners pattern from view-registry.ts, but specialized for CodeMirror extensions.

```typescript
// Key type definitions:
interface GutterExtensionEntry {
  /** Qualified ID: "{pluginId}.{decorationId}" */
  id: string;
  /** The CodeMirror Extension object (result of calling the factory) */
  extension: Extension;
}

// Module-level state:
const gutterMap = new Map<string, GutterExtensionEntry>();
const listeners = new Set<() => void>();
let snapshot: Extension[] = [];

// Public API:
export function registerGutterExtension(entry: GutterExtensionEntry): () => void;
export function getGutterExtensions(): Extension[];
export function onGutterExtensionChange(listener: () => void): () => void;
export function useGutterExtensions(): Extension[];
```

- `registerGutterExtension(entry)`: Adds entry to map, rebuilds snapshot (flat array of all Extension objects sorted by insertion order), notifies listeners. Returns dispose function. Throws if id already registered.
- `getGutterExtensions()`: Returns the snapshot array. Returns frozen empty array `[]` when no extensions registered (same pattern as view-registry).
- `onGutterExtensionChange(listener)`: Subscribe pattern. Returns unsubscribe.
- `useGutterExtensions()`: React hook via `useSyncExternalStore(onGutterExtensionChange, getGutterExtensions)`.

Import `Extension` type from `@codemirror/state`. Import `useSyncExternalStore` from `react`.

**Changes:**

1. Import `Compartment` from `@codemirror/state`.
2. Import `useGutterExtensions` from `@/lib/plugins/gutter-extension-registry`.

3. Create a `Compartment` ref for plugin gutter extensions:

```typescript
const gutterCompartmentRef = useRef(new Compartment());
```

4. Get current gutter extensions via the hook:

```typescript
const pluginGutterExtensions = useGutterExtensions();
```

5. In the `extensions` useMemo array, add the compartment with current gutter extensions:
```typescript
base.push(gutterCompartmentRef.current.of(pluginGutterExtensions));
```
Add `pluginGutterExtensions` to the useMemo dependency array.

6. Add a `useEffect` that recompartmentalizes when gutter extensions change WITHOUT recreating the editor. This is the key part -- when plugins are installed/uninstalled, the editor should reconfigure rather than remount:
```typescript
useEffect(() => {
  const view = viewRef.current;
  if (!view) return;
  view.dispatch({
    effects: gutterCompartmentRef.current.reconfigure(pluginGutterExtensions),
  });
}, [pluginGutterExtensions]);
```

This effect runs when the plugin gutter extensions array changes (due to useSyncExternalStore triggering a re-render), and it reconfigures the Compartment in-place. The editor is NOT destroyed/recreated -- the Compartment handles the dynamic reconfiguration.

**Important:** The `gutterCompartmentRef` should persist across renders (useRef), while the extensions it wraps come from the hook. The Compartment.of() call in the initial extensions array sets up the initial state. The useEffect handles subsequent updates.

**Do NOT add pluginGutterExtensions to the main extensions useMemo dependencies** -- that would cause the entire editor to be destroyed and recreated. Instead, only include the static `gutterCompartmentRef.current.of([])` in the useMemo (with an empty array as initial value), and let the useEffect handle dynamic updates. This is the correct Compartment pattern.

Revised approach:
- In useMemo: `base.push(gutterCompartmentRef.current.of([]))`  (no new deps needed)
- In useEffect (after editor creation): dispatch reconfigure with current pluginGutterExtensions
- In separate useEffect (for subsequent updates): dispatch reconfigure when pluginGutterExtensions changes
