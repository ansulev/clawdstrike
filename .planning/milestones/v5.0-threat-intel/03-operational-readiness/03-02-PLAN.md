           - permalink: "https://otx.alienvault.com/indicator/{type}/{value}"
           - cacheTtlMs: 1800000 (30 min)

       - GET https://otx.alienvault.com/api/v1/user/me with X-OTX-API-KEY header

  <action>
    Create MISP plugin in `misp-plugin.ts`:

    1. Define the plugin manifest with:
       - id: "clawdstrike.misp"
       - name: "misp-intel", displayName: "MISP"
       - trust: "internal", categories: ["intel"]
       - activationEvents: ["onStartup"]
       - contributions.threatIntelSources: [{ id: "misp", name: "MISP", description: "Malware Information Sharing Platform (self-hosted)", entrypoint: "..." }]
       - requiredSecrets: [
           { key: "api_key", label: "MISP API Key", description: "Authentication key from your MISP instance" },
           { key: "base_url", label: "MISP Instance URL", description: "Base URL of your MISP instance (e.g., https://misp.example.com)" }
         ]

    2. Implement the ThreatIntelSource adapter:
       - supportedIndicatorTypes: ["ip", "domain", "url", "hash", "email"]
       - rateLimit: { requestsPerMinute: 30 } (conservative for self-hosted)
       - enrich(indicator, context):
         - Retrieve base_url from context.secrets.get("base_url") -- fail with clear error if not configured
         - POST {baseUrl}/attributes/restSearch
         - Header: `Authorization: {apiKey}`, `Content-Type: application/json`, `Accept: application/json`
         - Body: { "value": indicator.value, "returnFormat": "json" }
         - Map response to EnrichmentResult:
           - Count distinct events containing matching attributes
           - verdict.classification: > 3 events = "malicious", 1-3 = "suspicious", 0 = "unknown"
           - verdict.confidence: min(eventCount / 5, 0.9) (cap at 0.9 since MISP data quality varies)
           - verdict.summary: "Found in X MISP events across Y attributes" or "No matching attributes"
           - rawData: { matchedAttributes: count, eventCount, categories: unique attribute categories, threatLevel: highest event threat_level_id }
           - mitreTechniques: extract from event Galaxy tags matching "mitre-attack-pattern" (e.g., "T1059")
           - relatedIndicators: extract other attributes from matching events (different type/value from query), up to 10
           - permalink: null (instance-specific, could construct "{baseUrl}/events/index" but not guaranteed)
           - cacheTtlMs: 900000 (15 min -- MISP data can update frequently)

    3. Implement healthCheck():
       - GET {baseUrl}/servers/getVersion with Authorization header
       - Return true on 200

    MISP-specific concerns:
    - Base URL must NOT have trailing slash (strip if present)
    - Self-hosted means response time varies widely -- timeouts should be generous (10s)
    - TLP markings in MISP tags could map to shareability but defer to Phase 4
