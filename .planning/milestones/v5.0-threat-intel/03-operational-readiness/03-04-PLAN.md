<objective>
Implement auto-enrichment that automatically queues new findings for enrichment when they exceed a configurable confidence threshold, without requiring manual "Run Enrichment" button clicks.

Purpose: Operators dealing with high-volume finding streams need enrichment to happen automatically. Manual enrichment doesn't scale -- auto-enrichment ensures every significant finding gets contextualized by threat intel sources without operator intervention.

Output: AutoEnrichmentManager class with per-sentinel and per-source configuration, integrated with the finding store's new-finding dispatch path.
</objective>

         enabled: boolean;

         confidenceThreshold: number;
         /** Source IDs to auto-enrich with. Empty = all configured sources. */

       - Constructor takes: { orchestrator, extractIndicators, config? }

         - config: initial AutoEnrichmentConfig (defaults to disabled)

       - processNewFinding(finding: Finding): void
         - Guard: if not enabled, return immediately
         - Guard: if confidence < confidenceThreshold, return
         - Guard: if enabledSentinels !== "all" and finding.createdBy not in enabledSentinels, return
         - Debounce: if finding.id was processed in last 100ms, skip (prevent duplicate processing)
         - Extract indicators from finding
         - Filter to enabled sources (if enabledSources is non-empty)
         - Skip sources that already appear in finding.enrichments
         - Queue via orchestrator.enrich(finding.id, indicators, { sourceFilter })
         - Fire-and-forget (no await) -- enrichment results flow back via store dispatch

       - getConfig(): AutoEnrichmentConfig
       - updateConfig(partial: Partial<AutoEnrichmentConfig>): void
         - Merges with current config
         - Persists to localStorage key "clawdstrike:auto-enrichment-config"

       - destroy(): void -- cleans up debounce timers
