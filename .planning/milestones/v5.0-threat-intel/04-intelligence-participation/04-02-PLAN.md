AbuseIPDB v2 Report API (external):
```
POST https://api.abuseipdb.com/api/v2/report
Headers: Key: {api_key}, Accept: application/json
Body: { ip: string, categories: string, comment: string }
Response: { data: { ipAddress: string, abuseConfidenceScore: number } }
```

MISP REST API (external):
```
POST {base_url}/events/add
Headers: Authorization: {api_key}, Accept: application/json, Content-Type: application/json
Body: { Event: { info: string, distribution: number, threat_level_id: number, analysis: number, Attribute: [{ type: string, category: string, value: string }] } }
Response: { Event: { id: string, info: string } }
```

  <behavior>
    AbuseIPDB reporting:
    - reportToAbuseIPDB with valid IP, apiKey, categories, comment calls fetch to "https://api.abuseipdb.com/api/v2/report" with POST method
    - Request includes headers: { Key: apiKey, Accept: "application/json", "Content-Type": "application/json" }
    - Request body includes { ip, categories (comma-joined), comment }
    - Returns { success: true, data: responseData } on 200 response
    - Returns { success: false, error: "message" } on non-200 response
    - Returns { success: false, error: "message" } on fetch error (network failure)

    MISP reporting:
    - reportToMisp with valid baseUrl, apiKey, indicator, eventInfo calls fetch to "{baseUrl}/events/add"
    - Request includes headers: { Authorization: apiKey, Accept: "application/json", "Content-Type": "application/json" }
    - Request body includes Event object with info, distribution=0 (org only), threat_level_id derived from severity, analysis=2 (completed), and Attribute array with the indicator
    - Maps iocType to MISP attribute type: "ip" -> "ip-dst", "domain" -> "domain", "sha256"/"sha1"/"md5" -> "sha256"/"sha1"/"md5", "url" -> "url", "email" -> "email-src"
    - Returns { success: true, eventId: string } on 200 response
    - Returns { success: false, error: "message" } on failure
  </behavior>
  <action>
    1. Create `threat-reporting.ts`:
       - `AbuseIPDBReportPayload` interface: `{ ip: string; categories: number[]; comment: string }`
       - `MispEventPayload` interface: `{ indicator: string; iocType: string; eventInfo: string; severity: "low" | "medium" | "high" | "critical" }`
       - `ReportResult` type: `{ success: true; data?: unknown } | { success: false; error: string }`
       - `reportToAbuseIPDB(payload: AbuseIPDBReportPayload, apiKey: string): Promise<ReportResult>`:
         - POST to `https://api.abuseipdb.com/api/v2/report`
         - Headers: `Key: {apiKey}`, `Accept: application/json`, `Content-Type: application/json`
         - Body: `{ ip, categories: categories.join(","), comment }`
         - Parse response, return success/failure
       - `reportToMisp(payload: MispEventPayload, apiKey: string, baseUrl: string): Promise<ReportResult>`:
         - POST to `{baseUrl}/events/add`
         - Headers: `Authorization: {apiKey}`, `Accept: application/json`, `Content-Type: application/json`
         - Body: MISP Event JSON with info, distribution=0, threat_level_id mapped from severity (low=3, medium=2, high=1, critical=1), analysis=2, Attribute array with type mapped from iocType
         - `mapIocTypeToMispAttrType(iocType: string)`: maps "ip"->"ip-dst", "domain"->"domain", "sha256"->"sha256", "sha1"->"sha1", "md5"->"md5", "url"->"url", "email"->"email-src", default->"text"
         - Parse response, return success with eventId or failure

    2. Write tests with vitest. Mock global `fetch` using `vi.stubGlobal`. Test both success and error paths for each function. Test the iocType-to-MISP-attribute mapping. Test that headers include the correct authentication format for each service.
  </action>
