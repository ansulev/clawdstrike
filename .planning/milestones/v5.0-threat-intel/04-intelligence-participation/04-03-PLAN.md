  <behavior>
    aggregateIndicators:
    - Given 3 findings where finding A and C both have IP "1.2.3.4" in their ioc_extraction enrichments, returns an IndicatorAggregation with indicator "1.2.3.4" having findingIds: [A.id, C.id] and count: 2
    - Deduplicates by (iocType, indicator) tuple
    - Sorts by count descending (most cross-referenced first)
    - Returns empty array when no findings have ioc_extraction enrichments

    aggregateVerdictsBySource:
    - Given findings enriched by "virustotal" and "greynoise", groups enrichments by source name
    - For each source, counts enrichments by verdict classification (from data.verdict or data.classification): malicious, benign, unknown, suspicious
    - Returns a map of sourceId -> { total, malicious, benign, unknown, suspicious }
    - Handles enrichments without verdict data gracefully (counts as "unknown")

    getSourceHealthSummary:
    - Given a list of source configs (id, name, rateLimit, quotaUsed, quotaTotal, lastSuccess, lastError), returns SourceHealthStatus[] with computed health: "healthy" (lastSuccess recent, quota < 80%), "degraded" (quota 80-95% or errors in last hour), "unhealthy" (quota > 95% or no successful call in last hour)
    - Includes quotaPercent computed from quotaUsed/quotaTotal
  </behavior>
  <action>
    1. Create `enrichment-aggregator.ts`:
       - `IndicatorAggregation` interface: `{ indicator: string; iocType: string; findingIds: string[]; findingTitles: string[]; count: number; lastSeenAt: number }`
       - `VerdictSummary` interface: `{ sourceId: string; sourceName: string; total: number; malicious: number; benign: number; unknown: number; suspicious: number }`
       - `SourceHealthInput` interface: `{ id: string; name: string; rateLimit: { maxPerMinute: number }; quotaUsed: number; quotaTotal: number; lastSuccessAt: number | null; lastErrorAt: number | null; lastErrorMessage?: string }`
       - `SourceHealthStatus` interface: `{ id: string; name: string; health: "healthy" | "degraded" | "unhealthy"; quotaPercent: number; quotaUsed: number; quotaTotal: number; rateLimit: { maxPerMinute: number }; lastSuccessAt: number | null; lastErrorAt: number | null; lastErrorMessage?: string }`

       - `aggregateIndicators(findings: Finding[]): IndicatorAggregation[]`:
         - Iterate findings, find enrichments of type "ioc_extraction"
         - Extract indicators from enrichment.data.indicators (typed as { indicator: string; iocType: string }[])
         - Build a Map keyed by `${iocType}:${indicator}` accumulating finding IDs and titles
         - Convert to array, sort by count desc, return

       - `aggregateVerdictsBySource(findings: Finding[]): VerdictSummary[]`:
         - Iterate all enrichments across all findings
         - Group by enrichment.source
         - For each enrichment, read `data.verdict` or `data.classification` as string, map to malicious/benign/unknown/suspicious bucket
         - Return sorted by total desc

       - `getSourceHealthSummary(sources: SourceHealthInput[]): SourceHealthStatus[]`:
         - For each source, compute quotaPercent = (quotaUsed / quotaTotal) * 100 (or 0 if quotaTotal is 0)
         - Determine health:
           - "unhealthy": quotaPercent > 95 OR (lastErrorAt is within last hour AND lastSuccessAt is null or older than lastErrorAt)
           - "degraded": quotaPercent > 80 OR lastErrorAt is within last hour
           - "healthy": otherwise

    2. Write tests with vitest. Create mock findings with ioc_extraction enrichments containing indicators arrays. Test cross-finding deduplication and counting. Test verdict aggregation with various data shapes. Test health thresholds.
  </action>
