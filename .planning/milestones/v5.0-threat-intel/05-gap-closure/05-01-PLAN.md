# Phase 5 Plan 1: Plugin Bootstrap, Mount FindingDetailActions and EnrichmentDashboard

## Objective

Wire the enrichment pipeline end-to-end by closing three gaps identified in the v5.0 audit:
1. The 6 threat intel plugin manifests are declared but never registered at app startup
2. FindingDetailActions (with "Report to..." button) is not mounted in finding-detail.tsx
3. EnrichmentDashboard is not accessible from any route or tab

## Context

- @apps/workbench/src/lib/plugins/threat-intel/ -- 6 plugin files with manifests and factory functions
- @apps/workbench/src/lib/workbench/threat-intel-registry.ts -- registerThreatIntelSource API
- @apps/workbench/src/components/workbench/findings/finding-detail.tsx -- needs FindingDetailActions
- @apps/workbench/src/components/workbench/findings/finding-detail-actions.tsx -- orphaned component
- @apps/workbench/src/components/workbench/findings/enrichment-dashboard.tsx -- orphaned component
- @apps/workbench/src/components/workbench/findings/findings-intel-page.tsx -- tab container for dashboard
- @apps/workbench/src/lib/workbench/secure-store.ts -- secureStore for API key retrieval
- @apps/workbench/src/lib/plugins/secrets-api.ts -- createSecretsApi for plugin-scoped keys

## Tasks

### Task 1: Create threat intel plugin bootstrap module
type="auto"

Create `apps/workbench/src/lib/plugins/threat-intel/bootstrap.ts` that:
1. Imports all 6 manifests (VIRUSTOTAL_MANIFEST, GREYNOISE_MANIFEST, SHODAN_MANIFEST, ABUSEIPDB_MANIFEST, OTX_MANIFEST, MISP_MANIFEST)
2. Imports all 6 factory functions (createVirusTotalSource, createGreyNoiseSource, createShodanSource, createAbuseIPDBSource, createOtxSource, createMispSource)
3. Imports registerThreatIntelSource from threat-intel-registry
4. Imports secureStore to read API keys at registration time
5. Exports a `bootstrapThreatIntelPlugins()` async function that:
   - For each plugin, reads the API key from secureStore using the `plugin:{pluginId}:api_key` key pattern
   - If a key exists, creates the source via the factory and registers it
   - For MISP, also reads `plugin:clawdstrike.misp:base_url`
   - Logs which plugins were registered vs skipped (no key)
   - Returns the count of registered sources
   - Uses try/catch per-plugin so one failure doesn't block others

Done when: Module exports `bootstrapThreatIntelPlugins` and handles all 6 plugins.

### Task 2: Wire bootstrap into App startup
type="auto"

Import and call `bootstrapThreatIntelPlugins()` from the App.tsx `useEffect` that already initializes secureStore. The bootstrap must run after `secureStore.init()` completes.

Done when: `bootstrapThreatIntelPlugins()` is called in App.tsx during startup.

### Task 3: Mount FindingDetailActions in finding-detail.tsx
type="auto"

Replace the inline action buttons in finding-detail.tsx's header with the `FindingDetailActions` component. This component already renders the same Confirm/Dismiss/Promote/Mark FP buttons plus adds the "Report to..." button for confirmed findings. Wire the `getApiKey` prop to read from secureStore using the plugin key pattern.

Done when: FindingDetailActions is rendered in finding-detail.tsx header with getApiKey wired to secureStore.

### Task 4: Mount EnrichmentDashboard as a tab in FindingsIntelPage
type="auto"

Add a third "Dashboard" tab to the FindingsIntelPage tab bar (alongside "Findings" and "Intel"). When active, render the EnrichmentDashboard component with findings from the store.

Done when: EnrichmentDashboard is accessible via a "Dashboard" tab on the findings page.
