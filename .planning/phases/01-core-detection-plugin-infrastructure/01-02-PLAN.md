Create `field-mappings.ts` with:

1. `FieldMappingEntry` interface:
```typescript
export interface FieldMappingEntry {
  /** Sigma canonical field name (e.g. "CommandLine"). */
  sigmaField: string;
  /** Splunk CIM equivalent (e.g. "process"). */
  splunkCIM?: string;
  /** Microsoft Sentinel equivalent (e.g. "CommandLine"). */
  sentinelField?: string;
  /** Elastic Common Schema equivalent (e.g. "process.command_line"). */
  ecsField?: string;
  /** Google UDM path equivalent (e.g. "target.process.command_line"). */
  udmPath?: string;
  /** Category for grouping in UI (e.g. "process", "file", "network", "dns", "registry", "authentication"). */
  category: string;
}
```

2. `BUILTIN_FIELD_MAPPINGS: FieldMappingEntry[]` constant with 50+ entries covering these categories:

**Process (15+ entries):**
CommandLine, Image (process path), ParentImage, ParentCommandLine, User, IntegrityLevel, ProcessId, ParentProcessId, CurrentDirectory, OriginalFileName, Product, Company, Description, Hashes, LogonId

**File (8+ entries):**
TargetFilename, SourceFilename, CreationUtcTime, Contents, FilePath, FileExtension, FileSize, FileOwner

**Network (10+ entries):**
SourceIp, DestinationIp, SourcePort, DestinationPort, Protocol, DestinationHostname, SourceHostname, Initiated, DestinationIsIpv6, QueryName

**DNS (5+ entries):**
QueryName, QueryType, QueryStatus, QueryResults, record_type

**Registry (5+ entries):**
TargetObject, Details, EventType, NewValue, OldValue

**Authentication (7+ entries):**
LogonType, AuthenticationPackageName, TargetUserName, TargetDomainName, SourceWorkstation, IpAddress, SubjectUserName

Use the mapping data from the research doc Section 5.4 as reference. For fields not in the research, use best-effort mappings documented with a JSDoc `/** @confidence approximate */` comment on uncertain entries.

3. Module-level `Map<string, FieldMappingEntry>` as backing store, keyed by `sigmaField`. Auto-populate from `BUILTIN_FIELD_MAPPINGS` at module load.

4. Registry functions:
- `registerFieldMappings(entries: FieldMappingEntry[]): () => void` -- adds entries to the map (keyed by sigmaField), returns dispose function. If a sigmaField already exists, the new entry MERGES into the existing one (fills in undefined platform fields without overwriting existing ones). This allows plugins to extend mappings.
- `getFieldMapping(sigmaField: string): FieldMappingEntry | null` -- returns entry or null.
- `translateField(sigmaField: string, targetFormat: "splunkCIM" | "sentinelField" | "ecsField" | "udmPath"): string | null` -- convenience lookup that returns the mapped field name for a given platform, or null if unmapped.
- `getAllFieldMappings(): FieldMappingEntry[]` -- returns all entries.
- `getFieldMappingsByCategory(category: string): FieldMappingEntry[]` -- returns entries filtered by category.
