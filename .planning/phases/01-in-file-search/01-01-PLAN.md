CRITICAL INTERACTION: The `ShortcutProvider` (shortcut-provider.tsx) listens on `window` with `capture: true`. It intercepts keydown events BEFORE CodeMirror processes them. If `edit.find` has `keybinding: "Meta+F"`, the ShortcutProvider will intercept Cmd+F, call `e.preventDefault()`, and execute the command handler.    **IMPORTANT CORRECTION** for the replace command: After investigation, the correct approach for opening the search panel in replace mode in CodeMirror 6 is to find and run the Mod-h handler from `searchKeymap`. Here is the actual implementation to use:

   ```typescript
   execute: () => {
     const view = getActiveEditorView();
     if (!view) return;
     // Find the Mod-h command from searchKeymap which opens search with replace enabled
     const replaceCmd = searchKeymap.find(
       (k) => k.key === "Mod-h" || k.key === "Mod-H"

     if (replaceCmd?.run) {
       replaceCmd.run(view);
     } else {
       // Fallback: open find-only panel
       openSearchPanel(view);
     }
   },
   ```
