Output: A working CLI that can generate project directories with config files (the type-specific source templates come in Plan 02).

1. Create `packages/cli/create-plugin/` directory.

2. Create `packages/cli/create-plugin/src/types.ts` with shared types:
   - `PluginType = "guard" | "detection" | "ui" | "intel" | "compliance" | "full"`
   - `ContributionPoint = "guards" | "commands" | "fileTypes" | "detectionAdapters" | "threatIntelSources" | "complianceFrameworks" | "editorTabs" | "bottomPanelTabs" | "rightSidebarPanels" | "statusBarItems" | "activityBarItems"`
   - `ScaffoldOptions = { name: string; displayName: string; publisher: string; type: PluginType; contributions: ContributionPoint[]; packageManager: "npm" | "bun" | "pnpm"; outputDir: string }`
   - `PLUGIN_TYPE_DEFAULTS: Record<PluginType, ContributionPoint[]>` mapping each type to its default contributions (e.g., "guard" -> ["guards", "commands"], "detection" -> ["detectionAdapters", "fileTypes", "commands"], "ui" -> ["editorTabs", "commands", "activityBarItems"], "intel" -> ["threatIntelSources", "commands"], "compliance" -> ["complianceFrameworks", "commands"], "full" -> all contribution points)

3. Create `packages/cli/create-plugin/src/flags.ts`:
   - Export `parseFlags(argv: string[]): ScaffoldOptions | null`
   - Parse process.argv for: positional `name`, `--type <type>`, `--contributions <comma-list>`, `--publisher <name>`, `--pm <npm|bun|pnpm>`, `--non-interactive`
   - Validate name is kebab-case via regex `/^[a-z][a-z0-9]*(-[a-z0-9]+)*$/`
   - Validate type is one of the 6 PluginType values
   - If contributions not provided, use PLUGIN_TYPE_DEFAULTS for the given type
   - Default publisher to "my-org"
   - Default packageManager to "npm"
   - Set outputDir to `path.resolve(process.cwd(), name)`
   - Derive displayName from name by capitalizing words (e.g., "my-guard" -> "My Guard")
   - Return null if required flags missing (name, type)

4. Create `packages/cli/create-plugin/src/prompts.ts`:
   - Export `runInteractivePrompts(initialName?: string): Promise<ScaffoldOptions>`
   - Use `@clack/prompts` (import { intro, outro, text, select, multiselect, confirm, isCancel, cancel, spinner })
   - Prompt sequence:
     a. `intro("Create ClawdStrike Plugin")`
     b. `text({ message: "Plugin name (kebab-case):", validate: kebab-case regex, initialValue: initialName })`
     c. `text({ message: "Display name:", initialValue: derived from name })`
     d. `text({ message: "Publisher:", initialValue: "my-org" })`
     e. `select({ message: "Plugin type:", options: [{ value: "guard", label: "Guard - Custom security guard" }, { value: "detection", label: "Detection - Detection format adapter" }, { value: "ui", label: "UI - Custom panels and views" }, { value: "intel", label: "Intel - Threat intelligence source" }, { value: "compliance", label: "Compliance - Compliance framework" }, { value: "full", label: "Full - All contribution points" }] })`
     f. `multiselect({ message: "Contribution points:", options: [...all ContributionPoint values as { value, label }], initialValues: PLUGIN_TYPE_DEFAULTS[selectedType] })`
     g. `select({ message: "Package manager:", options: [{ value: "npm" }, { value: "bun" }, { value: "pnpm" }] })`
   - Handle `isCancel()` on each prompt — call `cancel()` and `process.exit(0)`
   - Return assembled ScaffoldOptions

5. Create `packages/cli/create-plugin/src/cli.ts`:
   - Export `async function main(): Promise<void>`
   - Check for `--non-interactive` flag in process.argv
   - If non-interactive: call `parseFlags(process.argv.slice(2))`, exit with error message if null
   - If interactive: extract positional name arg, call `runInteractivePrompts(name)`
   - Call `scaffoldProject(options)` (imported from engine.ts — will be created in Task 2)
   - Print success message with next steps (cd, install, build, test commands)
