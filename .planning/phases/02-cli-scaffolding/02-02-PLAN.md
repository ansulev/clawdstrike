Purpose: Plugin authors get meaningful, type-safe boilerplate for their chosen plugin type -- not a blank canvas. Each template compiles under TypeScript strict mode.
Output: Complete template set that the scaffold engine (from Plan 01) writes to disk.

- Import `createPlugin` and relevant types from `@clawdstrike/plugin-sdk`

- Use `export default createPlugin({ manifest: {...}, activate(ctx) {...} })`
- Use the `options.name` as the manifest `name`, `options.displayName` as `displayName`, `options.publisher` as `publisher`
- Use `${options.publisher}.${options.name}` as the manifest `id`
- Set version to "0.1.0", trust to "community", categories based on type
- Fill in activationEvents: ["onStartup"]

1. Create `packages/cli/create-plugin/src/templates/guard.ts`:
   Export `guardSourceTemplate(options: ScaffoldOptions): string`
   Generated code should:
   - Declare a GuardContribution with id `${options.name}-guard`, name from options.displayName + " Guard", technicalName from options.name with hyphens replaced by underscores, description "Custom guard provided by ${options.displayName}", category "custom", defaultVerdict "deny", icon "shield", configFields with one example toggle field `{ key: "enabled", label: "Enabled", type: "toggle", defaultValue: true }`
   - In activate(): `ctx.guards.register(guardDef)` and push the disposable to `ctx.subscriptions`
   - Also register a command `${options.publisher}.${options.name}.configure` with title "Configure ${options.displayName}"
   - manifest.contributions should declare the guard and command arrays

2. Create `packages/cli/create-plugin/src/templates/detection.ts`:
   Export `detectionSourceTemplate(options: ScaffoldOptions): string`
   Generated code should:
   - Declare a FileTypeContribution with id from options.name, label options.displayName, shortLabel first 3 uppercase chars of name, extensions [".${options.name}"], iconColor "#4FC3F7", defaultContent with a template comment, testable true
   - Declare a DetectionAdapterContribution with fileType matching the FileTypeContribution id, entrypoint "./adapter"
   - In activate(): register the file type via `ctx.fileTypes.register(fileTypeDef)`, push disposable
   - Also register a "Validate ${options.displayName}" command
   - manifest.contributions should declare fileTypes, detectionAdapters, and commands

3. Create `packages/cli/create-plugin/src/templates/ui.ts`:
   Export `uiSourceTemplate(options: ScaffoldOptions): string`
   Generated code should:
   - Declare an EditorTabContribution with id "${options.name}-panel", label options.displayName, icon "layout-panel-top", entrypoint "./panel"
   - Declare an ActivityBarItemContribution with id "${options.name}-sidebar", section "plugins", label options.displayName, icon "puzzle", href "/${options.name}"
   - In activate(): register a command `${options.publisher}.${options.name}.open` with title "Open ${options.displayName}" that would open the editor tab
   - manifest.contributions should declare editorTabs, activityBarItems, and commands

Use template literal strings. Escape backticks in the generated code with \`. Use consistent 2-space indentation in generated output.
