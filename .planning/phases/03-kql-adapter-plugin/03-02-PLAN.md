**KqlVisualPanel component (main):**

Accept `DetectionVisualPanelProps`. Use `useMemo` to parse source via `parseKqlQuery(source)`.

Build a `reconstructKql()` callback that takes the parsed state (table, where clauses, project columns, extend expressions, comments) and reconstructs valid KQL text. Call `onSourceChange(reconstructed)` when any field changes.

Layout (inside ScrollArea):

1. **Source Table Section** (Section with IconDatabase):
   - SelectInput dropdown showing SENTINEL_TABLES. Value = parsed.tableName. If the parsed table name is not in the predefined list, show it as a custom entry at the top. On change: update table name in reconstructed KQL.
   - If readOnly, show plain text instead of dropdown.

2. **Filter Chain Section** (Section with IconFilter):
   - Each `KqlWhereClause` renders as a card/row with:
     - A TextInput for the field name (e.g. "CommandLine")
     - A SelectInput for the operator (from KQL_OPERATORS)
     - A TextInput for the value (the right-hand side)
     - A small remove button (X icon) if not readOnly
   - An "Add Filter" button at the bottom (if not readOnly) that appends a new empty where clause `{ field: "", operator: "contains", value: "", raw: "" }`.
   - Card styling: use `accentColor` for left border accent (`borderLeft: 3px solid {accentColor}26`), dark background (`bg-white/5`), rounded corners.
   - On any field/operator/value change: reconstruct KQL and call onSourceChange.

3. **Projection Section** (Section with IconColumns):
   - If `parsed.projectColumns.length > 0`, show a comma-separated list of projected columns in a TextInput.
   - On change: parse comma-separated values, reconstruct KQL.
   - If no project clause, show a muted hint: "No projection specified -- all columns returned".

4. **Extend Expressions Section** (conditionally rendered):
   - Only shown if `parsed.extendExpressions.length > 0`.
   - Display each extend expression in a read-only code block (extend expressions are complex; visual editing deferred).

5. **Raw KQL Preview** (Section with IconCode, collapsed by default):
   - A small collapsible section at the bottom showing the raw reconstructed KQL in a `<pre>` block with mono font.
   - Styled with muted text color, useful for debugging round-trip fidelity.

**Styling notes:**
- Follow sigma-visual-panel.tsx conventions for spacing, colors, and component usage.
- Use `accentColor` prop (with fallback to DEFAULT_ACCENT) for header borders and active element highlights.
- Use `style={{ borderColor: accentColor + "40" }}` for subtle accent borders (matching sigma panel pattern).
- Dark theme: `text-[#ece7dc]` for primary text, `text-[#6f7f9a]` for labels, `bg-[#1a1a2e]` for card backgrounds.

**Self-registration at module bottom:**
```typescript
registerVisualPanel("kql_rule", KqlVisualPanel);
export { KqlVisualPanel };
```

**Round-trip reconstruction logic (`reconstructKql`):**
1. Start with comments (lines starting with //).
2. Add table name on its own line.
3. For each where clause: `| where {field} {operator} "{value}"` (quote value unless operator is `in` -- then use `({value})`).
4. If project columns exist: `| project {columns.join(", ")}`.
5. For each extend expression: `| extend {expression}`.
6. Join with newlines.

**Part A: Create kql-adapter-plugin.ts example**

Create an example plugin manifest for the KQL adapter using `createPlugin()`, following the egress-guard-plugin.ts pattern:

```typescript
import { createPlugin } from "@clawdstrike/plugin-sdk";
import type { PluginContext } from "@clawdstrike/plugin-sdk";

export default createPlugin({
  manifest: {
    id: "clawdstrike.kql-adapter",
    name: "kql-adapter",
    displayName: "Microsoft Sentinel KQL Adapter",
    description: "First-class KQL detection format support with visual tabular expression editor and bidirectional Sigma translation.",
    version: "1.0.0",
    publisher: "clawdstrike",
    categories: ["detection"],
    trust: "internal",
    activationEvents: ["onFileType:kql_rule"],
    main: "./kql-adapter-plugin.ts",
    contributions: {
      detectionAdapters: [
        {
          fileType: "kql_rule",
          entrypoint: "./kql-adapter.ts",
          fileTypeDescriptor: {
            label: "Microsoft Sentinel KQL Rule",
            shortLabel: "KQL",
            extensions: [".kql"],
            iconColor: "#0078d4",
            defaultContent: "// KQL Detection Rule\nSecurityEvent\n| where EventID == 4688\n| where CommandLine contains \"powershell\"\n| project TimeGenerated, Computer, CommandLine",
            testable: true,
          },
          hasVisualPanel: true,
          translations: [
            { from: "sigma_rule", to: "kql_rule", lossless: true },
            { from: "kql_rule", to: "sigma_rule", lossless: false, lossDescription: "KQL aggregation (summarize), joins, time functions, and project clauses cannot be expressed in Sigma" },
          ],
        },
      ],
    },
  },
  activate(_ctx: PluginContext) {
    // The adapter, visual panel, and translation provider are self-registered
    // at module load via registerAdapter(), registerVisualPanel(), and
    // registerTranslationProvider(). The manifest above is declarative
    // documentation of those contributions for the plugin loader.
  },
});
```
