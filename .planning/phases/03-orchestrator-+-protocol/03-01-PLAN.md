Port ruflo's `agent-pool.ts` (476 lines) to `packages/swarm-engine/src/agent-pool.ts` with these mechanical transforms:

1. **No `extends EventEmitter`** -- TypedEventEmitter is injected via constructor parameter (same pattern as AgentRegistry):
```typescript
export class AgentPool {
  constructor(
    private readonly events: TypedEventEmitter<SwarmEngineEventMap>,
    config: Partial<AgentPoolConfig> = {},
  ) { ... }
}
```

2. **`this.emit('event', data)` -> `this.events.emit('kind', eventData)`** -- Map ruflo events to SwarmEngineEventMap event kinds. The pool emits pool-specific events that may not all be in the event map. For events like "agent.acquired" that have no SwarmEngineEvent kind, do NOT emit them -- these are internal pool operations. The pool does NOT emit agent.spawned or agent.terminated events (those are AgentRegistry's responsibility). Instead, rely on the orchestrator to mediate.

3. **`new Date()` -> `Date.now()`** for all timestamps. `PooledAgent.lastUsed` is `number` (Unix ms), not `Date`.

4. **`agent.id.id` -> flat string `agentId`** -- Pool tracks agents by string ID. Use `generateSwarmId('agt')` for internal agent creation in `createPooledAgent()`.

5. **`NodeJS.Timeout` -> `ReturnType<typeof setInterval> | null`** with null initialization.

6. **Remove `createDefaultCapabilities()`** -- Pool does not hardcode capabilities. The `createPooledAgent()` method creates a minimal agent placeholder with an agentId. The orchestrator or registry fills in capabilities.

7. **`getState()` returns `AgentPoolState`** using Record, not Map:
```typescript
getState(): AgentPoolState {
  const agents: Record<string, ...> = {};
  for (const [id, pooled] of this.pooledAgents) {
    agents[id] = { agentId: id, status: ..., lastUsed: pooled.lastUsed, ... };
  }
  return { config: { ...this.config }, agents, availableCount: ..., busyCount: ..., ... };
}
```

8. **`dispose()` is synchronous** -- only clears timers. Does NOT call shutdown(). Does NOT dispose shared emitter (per Phase 2 decision).

9. **Pool internal state uses Map** (for O(1) lookups) but `getState()` converts to Record:
```typescript
private pooledAgents = new Map<string, PooledAgent>();
private available = new Set<string>();
private busy = new Set<string>();
```

10. **Make acquire/release/add/remove synchronous** -- ruflo had them as async but pool manages logical state, not OS processes. Remove `async`/`await` from these methods since there is nothing to await (createPooledAgent is also synchronous in our port since we just generate an ID and create an object).
