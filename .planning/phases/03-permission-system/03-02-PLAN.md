<objective>
Add network domain scoping for fetch permissions, validate permissions in the manifest validator, wire permissions from manifest into the bridge host, and create a permission prompt for the install flow.

Purpose: Completes the permission system by adding domain-scoped network access control (so a VirusTotal plugin cannot exfiltrate to arbitrary domains), preventing installation of plugins with unknown permissions, and giving operators visibility into what a plugin can do before they install it.

Output: Network permission enforcement, manifest validation for permissions, plugin loader wiring, permission prompt in install flow.
</objective>

    **Part A: Network domain scoping in permissions.ts**

    1. Add `KNOWN_PERMISSIONS` constant: a Set containing all valid PluginPermission string values. Export it for use by the manifest validator.

    2. Add `checkNetworkPermission(url: string, allowedDomains: string[]): boolean` function:
       - Parse the URL to extract hostname. Use `new URL(url)` in try/catch -- return false on parse failure (fail-closed).
       - Check each allowedDomain pattern against the hostname:
         - Exact match: `hostname === domain`
         - Wildcard subdomain: if domain starts with `*.`, check `hostname.endsWith(domain.slice(1))` (e.g., `*.virustotal.com` matches `api.virustotal.com` but NOT `virustotal.com` itself)
       - Return true if any domain matches, false otherwise.

    3. Add `extractNetworkPermissions(permissions: (PluginPermission | NetworkPermission)[]): NetworkPermission[]` helper that filters out NetworkPermission objects from the permissions array.

    4. Add `networkPermissions` field to BridgeHostOptions (optional `NetworkPermission[]`).

    5. In PluginBridgeHost constructor, store networkPermissions. In `registerDefaultHandlers()`, add a `"network.fetch"` handler:
       - Extract url from params: `const { url, options } = params as { url: string; options?: RequestInit }`
       - Check networkPermissions: if no NetworkPermission entries exist, deny.
       - Call `checkNetworkPermission(url, networkPermission.allowedDomains)`. If denied, throw an error that will be caught by the handler dispatch and returned as PERMISSION_DENIED.
       - If allowed, perform `fetch(url, options)` on behalf of the plugin and return `{ status, statusText, headers: Object.fromEntries(response.headers), body: await response.text() }`.
       - NOTE: In tests, mock globalThis.fetch.

    6. Update METHOD_TO_PERMISSION to include `"network.fetch": "network:fetch"`.

         - If object with `type` field: validate it is "network:fetch", has `allowedDomains` as non-empty string array. If type is unknown, push error.
         - Otherwise: push error "must be a string permission or NetworkPermission object".

    8. Call `validatePermissions()` from `validateManifest()` when `input.permissions !== undefined`.

    **Part A: Plugin loader wiring**

    1. In `loadCommunityPlugin()`, extract permissions from manifest and pass to PluginBridgeHost:
       ```typescript
       // Extract simple permissions as string array
       const simplePermissions = (manifest.permissions ?? [])
         .filter((p): p is string => typeof p === 'string');

       // Extract network permissions
       const networkPerms = (manifest.permissions ?? [])
         .filter((p): p is NetworkPermission => typeof p === 'object' && p.type === 'network:fetch');

       const host = new PluginBridgeHost({
         pluginId,
         targetWindow,
         permissions: simplePermissions.length > 0 || networkPerms.length > 0
           ? simplePermissions
           : undefined, // no permissions field = allow all (backward compat)
         networkPermissions: networkPerms.length > 0 ? networkPerms : undefined,
       });
       ```

    IMPORTANT: When `manifest.permissions` is defined (even as empty array), pass it to the bridge host so enforcement activates. When it is `undefined` (old manifests without permissions field), do NOT pass permissions to maintain backward compatibility.

    Correct logic:
    ```typescript
    const perms = manifest.permissions;
    if (perms !== undefined) {
      // Manifest declares permissions (possibly empty) -- enforce
      const simplePerms = perms.filter(...);
      const netPerms = perms.filter(...);
      host = new PluginBridgeHost({ ..., permissions: simplePerms, networkPermissions: netPerms });
    } else {
      // Old manifest without permissions -- no enforcement
      host = new PluginBridgeHost({ ..., });
    }
    ```
