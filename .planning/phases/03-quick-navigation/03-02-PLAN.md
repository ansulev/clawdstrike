  <action>
Create `apps/workbench/src/features/navigation/breadcrumb-bar.tsx` exporting `BreadcrumbBar` component.

**Props:** `{ route: string }` -- the active pane view's route string (e.g. "/editor").

**Breadcrumb derivation logic:**
1. Read the active tab's `filePath` from `usePolicyTabsStore`. Find it by matching `state.activeTabId` to `state.tabs`.
2. Read the project name from `useProjectStore` (`state.project?.name`).
3. If `filePath` is null or the route is not "/editor", render nothing (return null) -- breadcrumbs only apply to editor tabs with file paths.
4. Split `filePath` on "/" to get path segments. The breadcrumb segments are: `[projectName, ...folderSegments, fileName]`.

**Rendering:**
- Height: `h-[24px]` -- compact bar, smaller than the tab bar.
- Background: same as tab bar `bg-[#0b0d13]`, bottom border `border-b border-[#1a1d27]`.
- Segments rendered as inline `<button>` elements with `text-[10px] font-mono text-[#6f7f9a]`.
- Separator between segments: `<IconChevronRight size={10} />` from `@tabler/icons-react` (already a project dependency) in `text-[#6f7f9a]/40`.
- Last segment (the file name) is highlighted: `text-[#ece7dc]`.
- Hover state on segment buttons: `hover:text-[#ece7dc] hover:bg-[#131721]` with `rounded px-1.5`.
- Left padding: `pl-4` to align with editor content.

**Click behavior:**
- Clicking the project name segment: no-op (or could toggle Explorer sidebar, but keep it simple -- no-op for now).
- Clicking a folder segment: call `useProjectStore.getState().actions.toggleDir(folderPath)` to expand that folder in the Explorer, ensuring the folder is visible. Also use the activity-bar store to ensure the Explorer sidebar is open if it isn't already. Check what store controls the active sidebar panel. If unclear, just call toggleDir -- the user can open Explorer manually.
- Clicking the file name segment: no-op (already viewing it).

**Edge cases:**
- If the project name is not available, use the first path segment or "Project" as fallback.
- If filePath has no folder segments (file at root), show just `[projectName, fileName]`.
