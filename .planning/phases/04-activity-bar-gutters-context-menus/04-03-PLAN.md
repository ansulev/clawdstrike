**1. Create `context-menu-registry.ts`:**

Follow the Map + snapshot + listeners pattern from view-registry.ts, specialized for context menu items.

```typescript
// Types:
export type ContextMenuTarget = "editor" | "sidebar" | "tab" | "finding" | "sentinel";

export interface ContextMenuItemRegistration {
  /** Qualified ID: "{pluginId}.{itemId}" */
  id: string;
  /** Display label for the menu item */
  label: string;
  /** Command ID to execute when clicked (looked up in command registry) */
  command: string;
  /** Optional icon identifier */
  icon?: string;
  /** Visibility predicate expression -- evaluated against workbench context */
  when?: string;
  /** Which context menu this item appears in */
  menu: ContextMenuTarget;
  /** Sort priority within the menu (lower = higher, default 100) */
  priority?: number;
}

// Module-level state:
const menuMap = new Map<string, ContextMenuItemRegistration>();
const listeners = new Set<() => void>();
let snapshotByMenu = new Map<ContextMenuTarget, ContextMenuItemRegistration[]>();

// Public API:
export function registerContextMenuItem(item: ContextMenuItemRegistration): () => void;
export function getContextMenuItems(): ContextMenuItemRegistration[];
export function getContextMenuItemsByMenu(menu: ContextMenuTarget): ContextMenuItemRegistration[];
export function onContextMenuChange(listener: () => void): () => void;
export function useContextMenuItems(menu: ContextMenuTarget): ContextMenuItemRegistration[];
```

- `registerContextMenuItem(item)`: Adds to map, rebuilds snapshots grouped by menu, notifies listeners. Returns dispose function. Throws on duplicate id.
- `getContextMenuItemsByMenu(menu)`: Returns snapshot for a specific menu target, sorted by priority. Returns frozen empty array when none.
- `useContextMenuItems(menu)`: React hook via useSyncExternalStore. Returns items for a specific menu.

**2. Add `evaluateWhenClause` function:**

A simple when-clause evaluator for visibility predicates. The `when` string is a simple expression language (matching VS Code's when-clause syntax subset):

Support these operators:
- Simple key existence: `"editorFocused"` -- true if context has that key and it's truthy
- Negation: `"!editorReadOnly"` -- true if key is falsy or absent
- Equality: `"fileType == 'clawdstrike_policy'"` -- true if context[key] equals value
- Inequality: `"fileType != 'yara_rule'"` -- true if not equal
- AND: `"editorFocused && fileType == 'clawdstrike_policy'"` -- both must be true

```typescript
export interface WhenContext {
  [key: string]: string | boolean | number | undefined;
}

export function evaluateWhenClause(when: string | undefined, context: WhenContext): boolean;
```

If `when` is undefined or empty string, return true (always visible).

The evaluator:
1. Split on `&&` to get clauses
2. For each clause, trim and check:
   - If starts with `!`: negate the rest as a key lookup
   - If contains `==`: split on `==`, compare context[key] to value (strip quotes)
   - If contains `!=`: split on `!=`, check inequality
   - Otherwise: treat as key lookup, truthy check
3. All clauses must be true (AND logic)

No `||` support needed for v1 -- keep it simple.

**1. Create `plugin-context-menu.tsx`:**

This component provides a reusable way for workbench context menus to include plugin-contributed items. It is a headless component (returns menu item elements, not a full menu) that consumers embed inside their context menu rendering.

```typescript
import { useContextMenuItems, evaluateWhenClause, type ContextMenuTarget, type WhenContext } from "@/lib/plugins/context-menu-registry";

interface PluginContextMenuItemsProps {
  /** Which context menu is being rendered */
  menu: ContextMenuTarget;
  /** Current workbench context for evaluating when-clauses */
  context: WhenContext;
  /** Callback when a plugin menu item is clicked. Receives the command ID. */
  onExecuteCommand: (commandId: string) => void;
  /** Optional className for the separator/group wrapper */
  className?: string;
}

export function PluginContextMenuItems({ menu, context, onExecuteCommand, className }: PluginContextMenuItemsProps) {
  const items = useContextMenuItems(menu);

  // Filter items by when-clause evaluation
  const visibleItems = items.filter(item => evaluateWhenClause(item.when, context));

  if (visibleItems.length === 0) return null;

  return (
    <>
      {/* Separator before plugin items */}
      <div className="h-px bg-[#2d3240] my-1" role="separator" />
      {visibleItems.map(item => (
        <button
          key={item.id}
          className="flex items-center gap-2 w-full px-3 py-1.5 text-left text-[11px] font-mono text-[#ece7dc] hover:bg-[#2d3240] rounded transition-colors"
          onClick={() => onExecuteCommand(item.command)}
        >
          {item.icon && (
            <span className="text-[#6f7f9a] text-xs w-4 text-center shrink-0">
              {/* Simple icon placeholder -- first char of icon name */}
              {item.icon.charAt(0).toUpperCase()}
            </span>
          )}
          <span className="truncate">{item.label}</span>
        </button>
      ))}
    </>
  );
}
```

Also export a `usePluginContextMenuItems` hook for consumers who want to build their own rendering:

```typescript
export function usePluginContextMenuItems(menu: ContextMenuTarget, context: WhenContext) {
  const items = useContextMenuItems(menu);
  return items.filter(item => evaluateWhenClause(item.when, context));
}
```
