  <action>
1. Create `packages/sdk/plugin-sdk/typedoc.json` with this configuration:
   - `"$schema": "https://typedoc.org/schema.json"`
   - `"entryPoints": ["src/index.ts"]`
   - `"out": "../../docs/book/api/plugin-sdk"` (relative from plugin-sdk package root, resolves to `docs/book/api/plugin-sdk/`)
   - `"plugin": ["typedoc-plugin-markdown"]` for markdown output compatible with mdBook linking
   - `"entryPointStrategy": "expand"`
   - `"exclude": ["**/node_modules/**", "**/*.test.ts"]`
   - `"excludePrivate": true`
   - `"excludeInternal": true`
   - `"hideGenerator": true`
   - `"readme": "none"` (no README duplication)
   - `"tsconfig": "./tsconfig.json"`

2. Update `packages/sdk/plugin-sdk/package.json`:
   - Add `"typedoc": "^0.28.0"` to devDependencies
   - Add `"typedoc-plugin-markdown": "^4.6.0"` to devDependencies
   - Add script `"docs": "typedoc"` to scripts

3. Append to `mise.toml` a new task block:
```toml
[tasks."docs:plugin-api"]
description = "Generate TypeDoc API reference for @clawdstrike/plugin-sdk"
run = """
cd packages/sdk/plugin-sdk && npm install --ignore-scripts && npx typedoc
"""
```

This task:
- Changes to the plugin-sdk directory
- Ensures deps are installed (for CI where node_modules may not exist)
- Runs typedoc which reads typedoc.json and outputs to docs/book/api/plugin-sdk/

Note: TypeDoc with typedoc-plugin-markdown generates .md files. The output lands in `docs/book/api/plugin-sdk/` which is the BUILD output directory (docs/book/), NOT the source directory (docs/src/). This is intentional -- the generated API docs are not mdBook source pages but standalone HTML/markdown that can be linked to from mdBook pages via relative paths to the build output.

Actually, since mdBook builds to `docs/book/` and TypeDoc also outputs there, they will coexist in the build directory. The API reference page in mdBook will link to the TypeDoc output using a relative path from the built book.
