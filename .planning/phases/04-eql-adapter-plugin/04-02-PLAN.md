<objective>
Build the EQL visual panel with a sequence builder for multi-event correlation. Single-event queries show a simpler condition editor. Sequence queries show step cards that can be added, removed, and reordered, with maxspan and until clause editing. All changes round-trip back to EQL source text.

Purpose: This is the differentiating visual experience for EQL -- no other adapter has multi-event sequence visualization. The sequence builder makes complex temporal correlation rules accessible to analysts who may not be fluent in EQL syntax.

Output: eql-visual-panel.tsx (registered via registerVisualPanel)
</objective>

**Constants:**
```typescript
const DEFAULT_ACCENT = "#f04e98"; // Elastic pink
const EVENT_CATEGORIES: EqlEventCategory[] = ["process", "file", "network", "registry", "dns", "any"];
const OPERATOR_OPTIONS = [
```

**Main component `EqlVisualPanel({ source, onSourceChange, readOnly, accentColor })`:**
1. Parse source with `useMemo(() => parseEql(source), [source])`
2. If parse fails, show a fallback message with the error
3. Branch on `ast.type`:
   - `"single"` -> render `<SingleQueryEditor>`
   - `"sequence"` -> render `<SequenceBuilder>`

**`SingleQueryEditor` subcomponent:**
- Props: `{ query: EqlSingleQuery, onUpdate: (q: EqlSingleQuery) => void, readOnly: boolean }`
- Renders:
  - **Event Category selector**: SelectInput with EVENT_CATEGORIES options
  - **Conditions list**: Each condition rendered as a row with:
    - Field name TextInput (placeholder: "process.name")
    - Operator SelectInput (from OPERATOR_OPTIONS)
    - Value TextInput (placeholder: '"value"' or '"*pattern*"')
    - Remove button (X icon, unless readOnly)
  - **Add Condition button**: Appends `{ field: "", operator: "==", value: "", negated: false }`
  - **Logic operator toggle**: "and" / "or" toggle between conditions
- On any change: clone query, update field, call `onUpdate(newQuery)`

**`SequenceBuilder` subcomponent:**
- Props: `{ query: EqlSequenceQuery, onUpdate: (q: EqlSequenceQuery) => void, readOnly: boolean }`
- Renders:
  - **Sequence Header Section:**
    - "by" fields: TextInput showing comma-separated fields (e.g. "host.id, user.name")
    - maxspan: TextInput (e.g. "5m", "1h", "30s"), optional
  - **Step Cards:** Each step is a bordered card showing:
    - Step number badge (1, 2, 3...) with accent-colored circle
    - Event category SelectInput
    - Conditions list (same ConditionRow component as SingleQueryEditor)
    - Move up / Move down buttons for reordering (unless first/last)
    - Remove step button (X icon)
    - Connecting line/arrow between steps (CSS vertical line + small arrow)
  - **Add Step button**: Appends a new step with `{ eventCategory: "process", conditions: [], logicOperator: "and" }`
  - **Until section (collapsible):** Optional "until" clause, renders as another event query card labeled "UNTIL" with a different border color (muted/gray)
- On any change: clone query, update, call `onUpdate(newQuery)`

**Round-trip mechanism:**
- When any editor callback fires, build new AST -> call `generateEql(newAst)` -> call `onSourceChange(newEqlText)`
- This ensures the source editor stays in sync

**ConditionRow reusable subcomponent:**
- Props: `{ condition: EqlCondition, onChange: (c: EqlCondition) => void, onRemove: () => void, readOnly: boolean }`
- Renders field, operator, value inputs in a horizontal row
- For "in" operator, show a comma-separated value input that splits into string[]

**Styling:**
- Use Tailwind classes consistent with sigma-visual-panel.tsx
- Accent color via `style={{ borderColor: accentColor }}` on section headers and step number badges
- Step cards: `bg-[#1a1a2e]/60 border border-white/10 rounded-lg p-3`
- Connecting lines between steps: `border-l-2` with accent color, 16px left margin

**Self-registration at module bottom:**
```typescript

export { EqlVisualPanel };
```
