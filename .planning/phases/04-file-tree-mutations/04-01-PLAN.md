    2. **Tauri bridge** (`tauri-bridge.ts`): Add three new async functions following the existing pattern (lazy import, `isDesktop()` gate, try/catch with console.error):

       - `createDetectionFile(dirPath: string, fileName: string, fileType: FileType): Promise<string | null>` — Builds the full path as `${dirPath}/${fileName}`, gets `defaultContent` from `FILE_TYPE_REGISTRY[fileType]`, calls `saveDetectionFile(defaultContent, fileType, fullPath)`. Returns the saved path or null.

       - `renameDetectionFile(oldPath: string, newPath: string): Promise<boolean>` — Lazy-imports `rename` from `@tauri-apps/plugin-fs`, calls `await rename(oldPath, newPath)`, returns true. On error, console.error and return false. Gate with `isDesktop()` returning false.

       - `deleteDetectionFile(filePath: string): Promise<boolean>` — Lazy-imports `remove` from `@tauri-apps/plugin-fs`, calls `await remove(filePath)`, returns true. On error, console.error and return false. Gate with `isDesktop()` returning false.

    3. **Project store** (`project-store.tsx`): Add three new actions to the `actions` object inside the Zustand store:

       - `createFile: async (parentDirPath: string, fileName: string, fileType: FileType) => Promise<string | null>` — Import `createDetectionFile` from tauri-bridge. Call it. On success, insert a new `ProjectFile` node into the tree under the matching parent directory (walk `project.files` recursively to find the dir with `path === parentDirPath`). Push to parent's `children` array. Re-sort children (dirs first, then alphabetical). Auto-expand the parent directory. Return the new file path. On failure, return null.

       - `renameFile: async (oldPath: string, newName: string) => Promise<boolean>` — Compute `newPath` by replacing the last path segment of `oldPath` with `newName`. Import `renameDetectionFile` from tauri-bridge. Call it. On success, walk the tree to find the node with `path === oldPath`, update its `name` to `newName`, update its `path` to `newPath`, and re-infer `fileType` via `inferFileTypeFromPath`. Return true. On failure return false.

       - `deleteFile: async (filePath: string) => Promise<boolean>` — Import `deleteDetectionFile` from tauri-bridge. Call it. On success, remove the node from the tree (walk parent's children to filter out `path === filePath`). Return true. On failure return false.

       Update the `ProjectStoreState` interface's `actions` type to include these three new methods. The `inferFileTypeFromPath` function is already in the file — keep it as-is and reuse it.

       For the tree mutation helpers: create a helper function `mutateTree(files: ProjectFile[], targetPath: string, mutator: (parent: ProjectFile[], index: number) => ProjectFile[]): ProjectFile[]` that recursively walks and applies the mutator when the target is found. This avoids duplicating recursive walk logic across create/rename/delete.
