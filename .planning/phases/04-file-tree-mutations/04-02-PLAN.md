    1. **Project store** (`project-store.tsx`): Add a `fileStatuses` map to track per-file status.
       - Add to `ProjectState` interface: `fileStatuses: Map<string, FileStatus>` where `FileStatus = { modified?: boolean; hasError?: boolean }`.
       - Export the `FileStatus` type.
       - Initialize `fileStatuses: new Map()` in the store default state.
       - Add two new actions:
         - `setFileStatus: (filePath: string, status: FileStatus) => void` — Sets or merges status for a file path in the map. `set({ fileStatuses: new Map(get().fileStatuses).set(filePath, { ...get().fileStatuses.get(filePath), ...status }) })`
         - `clearFileStatus: (filePath: string) => void` — Deletes the entry from the map. `const next = new Map(get().fileStatuses); next.delete(filePath); set({ fileStatuses: next })`

    2. **Explorer tree item** (`explorer-tree-item.tsx`): Add visual status indicators.
       - Add new props: `isModified?: boolean`, `hasError?: boolean`
       - **Modified dot**: When `isModified` is true AND the file is NOT a directory, render a small filled circle (4px) to the right of the file name, colored `#d4a84b` (gold, matching the project accent). Use `<span className="shrink-0 w-1 h-1 rounded-full bg-[#d4a84b] ml-auto" />` positioned at the right edge of the row (after the name span, using ml-auto to push right).
       - **Error badge**: When `hasError` is true AND the file is NOT a directory, render a small circle (4px) colored `#ef4444` (red-500). If both modified AND error, show only the error badge (error takes priority). Use `<span className="shrink-0 w-1 h-1 rounded-full bg-red-500 ml-auto" />`.
       - Also style the file name text: if `hasError`, add `text-red-400` class. If `isModified` (and not error), add `italic` class to the name span.

    4. **Sidebar panel** (`sidebar-panel.tsx`): Connect file statuses from store to Explorer.
