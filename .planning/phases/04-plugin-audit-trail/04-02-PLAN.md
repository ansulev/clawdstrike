  <behavior>
    - PluginReceiptForwarder.forward(receipt) adds receipt to internal queue
    - When hushd URL is configured and reachable, forward() sends a POST request to {hushdUrl}/api/v1/audit/plugin-receipts with the receipt as JSON body
    - When hushd is unreachable (fetch throws), receipt stays in the queue for retry
    - flush() processes all queued receipts, sending each to hushd, removing successful ones
    - When hushd returns non-2xx status, the receipt stays queued (retry on next flush)
    - PluginReceiptForwarder with hushdUrl = null silently drops forward calls (local-only mode)
    - createReceiptForwarder({ hushdUrl, authToken }) returns a configured forwarder
  </behavior>
  <action>
    **receipt-forwarder.ts:**
    Create PluginReceiptForwarder class:
    - Constructor takes { hushdUrl: string | null; authToken?: string }
    - Private queue: PluginActionReceipt[] for buffering when hushd is unavailable
    - forward(receipt: PluginActionReceipt): void -- if hushdUrl is null, return immediately (local-only mode). Otherwise, attempt POST to {hushdUrl}/api/v1/audit/plugin-receipts with JSON body. On success (2xx), done. On failure, push to queue for retry.
    - flush(): Promise<{ sent: number; failed: number }> -- iterate queue, attempt to send each, remove successes. Return counts.
    - getQueueSize(): number -- for monitoring
    - The POST request includes headers: Content-Type: application/json, Authorization: Bearer {authToken} (if provided)

    Create createReceiptForwarder(options: { hushdUrl: string | null; authToken?: string }): PluginReceiptForwarder factory function.

    Use global fetch (available in browser and modern Node). Wrap fetch in try/catch for network errors. Do NOT throw on failure -- audit forwarding is best-effort (fail-open for forwarding, fail-closed for receipt generation which is in Plan 01).

    **index.ts:**
    Add barrel re-exports for PluginReceiptForwarder, createReceiptForwarder.
  </action>
