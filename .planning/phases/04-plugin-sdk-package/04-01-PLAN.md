<objective>
Create the `@clawdstrike/plugin-sdk` TypeScript package that plugin authors import to get typed APIs for all contribution points, lifecycle hooks, and the `createPlugin()` factory.

Purpose: Plugin authors need a single import (`@clawdstrike/plugin-sdk`) that gives them full TypeScript autocompletion for manifest authoring, contribution point declaration, and the activate/deactivate lifecycle contract. This package is the public API surface of the plugin ecosystem.

Output: A publishable TypeScript package at `packages/sdk/plugin-sdk/` with types, `createPlugin()` helper, `PluginContext` interface, and tests.
</objective>

    export interface PluginContext {
      pluginId: string;
      subscriptions: Disposable[];
      commands: CommandsApi;
      guards: GuardsApi;
      fileTypes: FileTypesApi;
      statusBar: StatusBarApi;
      sidebar: SidebarApi;
      storage: StorageApi;
    }

    These are interfaces only -- the workbench PluginLoader creates concrete implementations at activation time. The SDK only defines the shape.

    **5. Create src/create-plugin.ts:**
    ```typescript
    export interface PluginDefinition {
      manifest: PluginManifest;
      activate(context: PluginContext): Disposable[] | void;
      deactivate?(): void;
    }

    export function createPlugin(definition: PluginDefinition): PluginDefinition {
      return definition;
    }
    ```

    The `createPlugin()` function is an identity function that provides type checking at the call site. When a plugin author writes `createPlugin({ manifest: {...}, activate(ctx) { ... } })`, TypeScript enforces that:

    - Return type is `Disposable[] | void`
    - `deactivate` is optional
