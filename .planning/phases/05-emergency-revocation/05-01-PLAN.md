<objective>
Build the client-side revocation infrastructure: a local revocation store, the "revoked" lifecycle state, bridge-level revocation checks, and plugin deactivation with graceful drain.

Purpose: This is the foundation for emergency revocation. It creates the mechanism for killing a plugin locally (store + state + bridge guard + deactivation), which Plan 02 wires to the hushd SSE event stream for fleet-wide coordination.

Output: PluginRevocationStore class, "revoked" lifecycle state, bridge host revocation guard, PluginLoader.revokePlugin() method with 5-second drain timeout.
</objective>
