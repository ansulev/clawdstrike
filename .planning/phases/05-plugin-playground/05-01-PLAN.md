    - `PlaygroundError` type: `{ message: string; line?: number; column?: number; stack?: string }`.
    - `ConsoleEntry` type: `{ level: "log" | "warn" | "error" | "info"; args: unknown[]; timestamp: number }`.
    - `ContributionSnapshot` type: `{ guards: string[]; commands: string[]; fileTypes: string[]; editorTabs: string[]; bottomPanelTabs: string[]; rightSidebarPanels: string[]; statusBarItems: string[] }`.
    - Export actions: `setSource(s: string)`, `setTranspiled(js: string | null)`, `setRunning(b: boolean)`, `incrementRunCount()`, `addError(e: PlaygroundError)`, `clearErrors()`, `addConsoleEntry(e: ConsoleEntry)`, `clearConsole()`, `setContributions(c: ContributionSnapshot | null)`.

    Create `apps/workbench/src/lib/plugins/playground/playground-transpiler.ts`:
    - Import `transform` from `sucrase` (already available in node_modules as transitive dep -- if import fails at runtime, fall back to stripping type annotations manually using a regex-based approach, but sucrase should work).
    - Export `transpilePlugin(source: string): { code: string; error: PlaygroundError | null }`.
    - Call `transform(source, { transforms: ["typescript", "imports"], filePath: "playground.tsx" })`.
    - The `"imports"` transform converts ESM imports to CJS requires. BUT for the playground we want ESM output. So only use `transforms: ["typescript"]` to strip types, keeping import/export syntax.
    - Wrap the transpiled code to create a module: prepend `const __exports = {};` and append `export default __exports;` -- actually no, the output should be a valid ES module that the dynamic import can consume. Just return the type-stripped JS as-is.
    - Rewrite `import { createPlugin } from "@clawdstrike/plugin-sdk"` to `import { createPlugin } from "/@id/@clawdstrike/plugin-sdk"` so Vite can resolve it. Actually, since the eval route serves from `/__plugin-eval/`, we need a different strategy: wrap the transpiled code so it does NOT use bare imports. Instead, inject `createPlugin` and SDK types as globals. The wrapper should be:
      ```
      const { createPlugin } = window.__CLAWDSTRIKE_PLUGIN_SDK__;
      // ... rest of transpiled code with imports stripped ...
      ```
    - So the transpiler must: (1) strip TypeScript types via sucrase with `transforms: ["typescript"]`, (2) replace `import { ... } from "@clawdstrike/plugin-sdk"` with destructuring from `window.__CLAWDSTRIKE_PLUGIN_SDK__`, (3) replace `export default` with `window.__PLAYGROUND_PLUGIN__ =`.
    - On sucrase error, return `{ code: "", error: { message, line, column } }` extracted from the error.

    Create `apps/workbench/src/lib/plugins/playground/playground-eval-server.ts`:
    - Export `pluginEvalMiddleware` -- a function compatible with Vite's `configureServer` middleware API: `(req: Connect.IncomingMessage, res: ServerResponse, next: Connect.NextFunction) => void`.
    - Handle `POST /__plugin-eval/` -- reads JSON body `{ code: string; runId: number }`, stores the code in a module-level Map keyed by runId, responds with `{ url: "/__plugin-eval/${runId}.js" }`.
    - Handle `GET /__plugin-eval/:runId.js` -- serves the stored code with `Content-Type: application/javascript` and CORS headers. This serves from `'self'` origin, avoiding CSP blob URL violations in Tauri.
    - Codes are evicted from the Map after 60 seconds (setTimeout cleanup).

    Update `apps/workbench/vite.config.ts`:
    - Import `pluginEvalMiddleware` from `./src/lib/plugins/playground/playground-eval-server`.
    - Add a custom Vite plugin in the `plugins` array (after `react()`):
      ```typescript
      {
        name: "clawdstrike-plugin-eval",
        configureServer(server) {
          server.middlewares.use("/__plugin-eval", pluginEvalMiddleware);
        },
      }
      ```
    - This ensures the `/__plugin-eval/` route is available during `vite dev`.

      11. Build `ContributionSnapshot` from the manifest's contributions and call `setContributions()`.
