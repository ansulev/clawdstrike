    - Return the rewritten stack trace string.
    - Also export `extractErrorLocation(stack: string): { line: number; column: number } | null` -- extracts the first `/__plugin-eval/` frame's line and column for editor gutter highlighting.

    Create `apps/workbench/src/components/plugins/playground/PluginConsolePanel.tsx`:
    - Import `usePlaygroundConsole` from playground store.
    - Render a list of `ConsoleEntry` items in a scrollable container.
    - Each entry shows: severity icon (Info/AlertTriangle/XCircle/MessageSquare from lucide-react), timestamp (HH:MM:SS.mmm format), and stringified args.
    - Severity icon colors: log=#c8d1e0 (default), info=#60a5fa (blue), warn=#fbbf24 (amber), error=#f87171 (red).
    - Severity filter: row of toggle buttons at the top for log/info/warn/error. Each button shows count in parentheses. Active filters have filled background, inactive are outline-only. All active by default.
    - Use `useState<Set<string>>` for active filter levels. Filter entries before rendering.
    - Clear button in the top-right calls `clearConsole()`.
    - Auto-scroll to bottom on new entries: `useEffect` with ref to scroll container, `scrollTo({ top: scrollHeight, behavior: "smooth" })`.
    - Stringify args: use `JSON.stringify(arg, null, 2)` for objects, `String(arg)` for primitives. Join multiple args with spaces.
    - Empty state: "No console output yet" with Terminal icon.
    - Console output MUST NOT leak into the global browser console. The playground-runner.ts already handles this via the __PLAYGROUND_CONSOLE__ proxy -- this panel just reads from the store.
    - Styling: monospace font for entries (`font-mono text-xs`), dark theme background.
    - Export as default and named `PluginConsolePanel`.

    Create `apps/workbench/src/components/plugins/playground/PlaygroundErrorBoundary.tsx`:
    - This is NOT a React error boundary (class component) -- it's a functional component that renders playground errors from the store.
    - Import `usePlaygroundErrors` from the store and `mapStackTrace`, `extractErrorLocation` from playground-source-map.ts.
    - Import `usePlaygroundSource` to access the current source for source mapping.
    - If errors array is non-empty, render an error overlay at the top of the playground editor pane (positioned absolutely, semi-transparent red background).
    - Show the first error's message prominently, with a "Dismiss" button that calls `clearErrors()`.
    - If the error has a `stack` property, call `mapStackTrace(stack, sourceLines)` and render the mapped stack trace in a pre/code block with monospace font.
    - If `extractErrorLocation(stack)` returns a line/column, display "Error at line {line}, column {column}" and highlight that information prominently.
    - The error line number should match what the user sees in the CodeMirror editor (since sucrase is line-preserving for type stripping).
    - Multiple errors: show a "1 of N errors" counter with prev/next navigation if more than one error exists.
    - Styling: error overlay uses `bg-[#1a0000]/95 border border-[#f87171]/30 rounded-lg p-4`, error message in `text-[#f87171] font-medium`, stack trace in `text-[#c8d1e0]/70 text-xs font-mono`.
    - Export as default and named `PlaygroundErrorBoundary`.

    Update `apps/workbench/src/components/plugins/playground/PlaygroundEditorPane.tsx`:
    - Import `PlaygroundErrorBoundary` and render it as an overlay layer within the editor pane (above the editor, using absolute positioning within a relative container).
    - The PlaygroundErrorBoundary should only render when there are errors, so the editor remains fully interactive underneath when no errors exist.
