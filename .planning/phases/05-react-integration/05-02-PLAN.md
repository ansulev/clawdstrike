  <action>
    Create the `apps/workbench/src/features/swarm/layout/` directory and two files.

    **topology-layout.ts** -- Pure-math module, no React imports:

    1. Export `LayoutResult` interface: `{ positions: Map<string, { x: number; y: number }> }`.

    2. Export `computeLayout(nodes, edges, topology, viewport)` function that dispatches to 4 internal algorithms based on `topology` parameter:

    **meshLayout (force-directed):**
    - Port `tickSimulation` logic from `apps/control-console/src/utils/forceLayout.ts` VERBATIM:
      - charge repulsion: `force = 500 / (dist * dist)`
      - spring attraction: `force = (dist - 80) * 0.01`
      - center gravity: `0.001`
      - damping: `0.9`
      - bounds clamping: `Math.max(radius, Math.min(width - radius, x))`
    - Use node radius of 60 (SwarmBoard node size)
    - Run simulation for 100 iterations (sufficient for convergence with damping=0.9)
    - Initialize node positions: spread evenly across viewport if no existing position, else use current position as starting point

    **hierarchicalLayout (Sugiyama-style):**
    - Layer assignment by nodeType: layer 0 = agentSession, layer 1 = terminalTask, layer 2 = artifact/receipt/diff/note
    - For nodes with edges: use BFS from roots (nodes with no incoming edges) to assign layers, taking max of (parent_layer + 1, type_layer)
    - Within each layer: sort by incoming edge source position (barycenter ordering)
    - Spacing: rankSep = 120 (vertical gap between layers), nodeSep = 80 (horizontal gap within layer)
    - Center each layer horizontally within viewport

    **centralizedLayout (hub-spoke):**
    - Find the hub node: first `agentSession` node, or first node if none
    - Place hub at viewport center: `(width/2, height/2)`
    - Arrange spoke nodes in a circle: `x = cx + radius * cos(2*PI*i/n)`, `y = cy + radius * sin(2*PI*i/n)`
    - Radius = `Math.min(width, height) * 0.35` (leaves margin)
    - Task/artifact nodes stack below their parent spoke at +120 y offset

    **hybridLayout (Sugiyama backbone + force within ranks):**
    - Use hierarchicalLayout for initial layer assignment and y-positioning
    - Within each rank (same y), run a simplified 1D force pass (charge repulsion + spring attraction on x-axis only, 50 iterations)
    - This gives hierarchical vertical structure with organic horizontal spacing

    For `topology === "adaptive"`, fall back to meshLayout.
