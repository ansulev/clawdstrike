Modify PaneTabBar to detect tab overflow and provide navigation controls:

1. Add a `useRef<HTMLDivElement>` for the scrollable tab container (the `div` with `overflow-x-auto scrollbar-hide`).

2. Add state: `const [canScrollLeft, setCanScrollLeft] = useState(false)` and `const [canScrollRight, setCanScrollRight] = useState(false)`.

3. Add a `checkOverflow` function that reads `scrollRef.current` and sets:
   - `canScrollLeft = scrollLeft > 0`
   - `canScrollRight = scrollLeft + clientWidth < scrollWidth - 1` (1px tolerance)

4. Use a `useEffect` with a `ResizeObserver` on `scrollRef.current` that calls `checkOverflow` on resize. Also attach a `scroll` event listener on the container to re-check on scroll. Clean up both on unmount.

5. Add an `onWheel` handler on the scrollable container div that calls `e.preventDefault()` and does `scrollRef.current.scrollLeft += e.deltaY` (converts vertical wheel to horizontal scroll). Only preventDefault when the container has overflow (scrollWidth > clientWidth).

6. Render left/right scroll arrow buttons:
   - Place them OUTSIDE the scrollable div but inside the tablist wrapper.
   - Left arrow: `IconChevronLeft` from `@tabler/icons-react`, size 14, stroke 1.8. Visible only when `canScrollLeft`. On click: `scrollRef.current.scrollBy({ left: -120, behavior: 'smooth' })`.
   - Right arrow: `IconChevronRight`, same pattern. Visible only when `canScrollRight`. On click: `scrollRef.current.scrollBy({ left: 120, behavior: 'smooth' })`.
   - Arrow button styling: `shrink-0 flex items-center justify-center w-6 text-[#6f7f9a] hover:text-[#ece7dc] hover:bg-[#131721] transition-colors` with border-r or border-l of `border-[#202531]`.

7. Layout order in the tablist row: [left-arrow (conditional)] [scrollable-tabs (flex-1 min-w-0)] [right-arrow (conditional)] [pane-action-buttons].

**Step A: Add batch-close store actions to pane-store.ts**

Add two new methods to the PaneStore interface and implementation:

1. `closeViewsToRight: (paneId: string, viewId: string) => void` -- finds the view's index in the pane's views array, then calls `closeView` for every view AFTER that index (iterate in reverse to avoid index shifting).

2. `closeSavedViews: (paneId: string) => void` -- In the pane system, PaneView has no `dirty` state (unlike PolicyTab), so "Close Saved" closes all tabs EXCEPT the active one (the active tab is the one being worked on, all others are "saved"). Implementation: iterate all views in the pane group, call `closeView` for each whose id !== the pane's `activeViewId`.

Also add: `closeOtherViews: (paneId: string, viewId: string) => void` -- close all views in the pane group except the one with the given viewId.

**Step B: Add context menu to PaneTabBar**

In pane-tab-bar.tsx, add context menu state and rendering following the pattern from policy-tab-bar.tsx:

1. Add state: `const [contextMenu, setContextMenu] = useState<{ viewId: string; paneId: string; x: number; y: number } | null>(null)`.

2. On each `PaneTab`, pass down an `onContextMenu` prop. In PaneTab, attach `onContextMenu` to the outer button element.

3. Create a `PaneTabContextMenu` component (inline in pane-tab-bar.tsx or pane-tab.tsx, whichever is cleaner) with these menu items:
   - "Close" -- calls `closeView(paneId, viewId)`
   - "Close Others" -- calls `closeOtherViews(paneId, viewId)`
   - "Close to the Right" -- calls `closeViewsToRight(paneId, viewId)`
   - separator
   - "Close Saved" -- calls `closeSavedViews(paneId)`
   - "Close All" -- closes all views: iterate pane.views and closeView each

4. Use icons from `@tabler/icons-react`: IconX for Close, IconArrowBarRight for Close to the Right, IconChecks for Close Saved, IconTrash for Close All. Size 12, stroke 1.5.

5. Style the context menu identically to policy-tab-bar.tsx:
   - Container: `fixed z-[100] min-w-[160px] bg-[#131721] border border-[#2d3240] rounded-md shadow-xl py-1`
   - Items: `flex items-center gap-2 w-full px-3 py-1.5 text-[11px] font-mono text-[#ece7dc] hover:bg-[#d4a84b]/10 hover:text-[#d4a84b] transition-colors text-left`
   - Separator: `h-px bg-[#2d3240] my-1`
   - Dismiss on mousedown outside or Escape key (useEffect with listeners, cleanup on unmount).

6. Update PaneTab to accept and forward the `onContextMenu` prop on its outer `<button>` element.
