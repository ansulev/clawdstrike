**Step A: Add store actions to bottom-pane-store.ts**

1. Add `splitTerminalIds: [string, string] | null` to BottomPaneStore state. Default: `null`. When non-null, the terminal panel renders two sessions side by side.

2. Add `renameTerminal: (id: string, title: string) => void` to BottomPaneStore interface and implementation. It maps over `terminalSessions`, finding the session with the matching id and updating its `title` field.

3. Add `splitTerminal: () => Promise<void>` to BottomPaneStore. Behavior:
   - If `splitTerminalIds` is already set, clear it (`set({ splitTerminalIds: null })`) -- this toggles split off.
   - If there are fewer than 2 terminal sessions, create a new one first via the existing `newTerminal()` flow.
   - Set `splitTerminalIds` to `[activeTerminalId, secondSessionId]` where secondSessionId is the first session that is NOT the active one.

4. Add `unsplitTerminal: () => void` -- sets `splitTerminalIds: null`.

5. Update `closeTerminal` -- if the closed session's id is in `splitTerminalIds`, set `splitTerminalIds: null` (exit split mode).

6. Add `splitTerminalIds: null` to `_reset`.

**Step B: Add split layout to terminal-panel.tsx**

1. Read `splitTerminalIds` from the store.

2. When `splitTerminalIds` is null (normal mode): render exactly as today -- single active session with a TerminalRenderer.

3. When `splitTerminalIds` is `[leftId, rightId]` (split mode):
   - Render a flex row (`flex flex-row h-full min-h-0`) containing two equal-width panels with a 1px vertical divider (`w-px bg-[#202531]`) between them.
   - Each panel: `flex-1 min-w-0 min-h-0` containing a TerminalRenderer for that session.
   - Look up the TerminalSession for each id. If a session's ptySessionId is null, show the error placeholder instead.

4. In the session tab bar (the row of session title buttons), highlight BOTH sessions that are in the split with the active style when in split mode.

**Step C: Add rename-on-double-click to terminal session tabs in terminal-panel.tsx**

1. Add state: `const [renamingId, setRenamingId] = useState<string | null>(null)`.

2. On the session title button, add `onDoubleClick={() => setRenamingId(session.id)}`.

3. When `renamingId === session.id`, render an inline input instead of the title text. Follow the RenameInput pattern from policy-tab-bar.tsx:
   - `useState` for value, `useRef<HTMLInputElement>` for the input element.
   - `useEffect` to `inputRef.current?.select()` on mount.
   - `onBlur` commits (call `renameTerminal(id, trimmedValue || currentTitle)` then `setRenamingId(null)`).
   - `onKeyDown`: Enter commits, Escape cancels (resets to original title).
   - Style: `bg-[#0b0d13] border border-[#d4a84b]/40 rounded px-1.5 py-0.5 text-[11px] font-mono text-[#ece7dc] outline-none focus:border-[#d4a84b] w-full max-w-[120px]`.

**Step D: Add split button to bottom-pane.tsx header**

1. In the bottom-pane.tsx header's right-side button group (where the "New terminal session" button is), add a "Split terminal" button BEFORE the new-terminal button. Only render when `activeTab === "terminal"`.

2. Use `IconLayoutColumns` from `@tabler/icons-react`, size 14, stroke 1.8. Same button styling as the existing new-terminal button.

3. On click: `useBottomPaneStore.getState().splitTerminal()`.

4. Title/aria-label: "Split terminal" when not split, "Unsplit terminal" when split. Read `splitTerminalIds` from store to determine state.
