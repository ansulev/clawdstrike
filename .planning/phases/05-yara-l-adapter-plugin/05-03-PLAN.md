Create `yaral-translation.ts` implementing bidirectional Sigma<->YARA-L translation.

**1. Sigma -> YARA-L Translation (`sigmaToYaral`):**

Input: Sigma YAML source string
Output: YARA-L rule text string

Steps:
- Parse Sigma YAML using `parseSigmaYaml()` from `../sigma-types`
- If parse fails, return error TranslationResult
- Map Sigma fields to UDM paths using `translateField(sigmaField, "udmPath")` from field-mappings.ts
- Build the YARA-L rule structure:

```
rule {sigmaTitle_normalized} {
  meta:
    author = "{rule.author || 'Detection Lab'}"
    description = "{rule.description || ''}"
    severity = "{sigmaLevelToChronicle(rule.level)}"
    created = "{rule.date || today}"
    {if rule.tags: mitre_attack = "{extractTechniques(tags).join(', ')}"}
    sigma_id = "{rule.id}"

  events:
    $e.metadata.event_type = "{logsourceToEventType(rule.logsource)}"
    {for each selection field:}
    {  $e.{udmPath} = "{value}" or $e.{udmPath} = /{value}/ nocase}

  condition:
    $e
}
```

- Sigma level mapping to Chronicle severity: critical->CRITICAL, high->HIGH, medium->MEDIUM, low->LOW, informational->INFORMATIONAL
- Sigma logsource category to UDM event_type:
  - process_creation -> PROCESS_LAUNCH
  - file_event/file_access -> FILE_CREATION
  - network_connection -> NETWORK_CONNECTION
  - dns/dns_query -> NETWORK_DNS
  - registry_* -> REGISTRY_MODIFICATION
  - authentication -> USER_LOGIN
  - default -> GENERIC_EVENT
- For Sigma modifiers:
  - `|contains` -> use regex: `$e.field = /value/ nocase`
  - `|startswith` -> use regex: `$e.field = /^value/ nocase`
  - `|endswith` -> use regex: `$e.field = /value$/ nocase`
  - `|re` -> use regex: `$e.field = /value/`
  - no modifier -> exact: `$e.field = "value"`
- Track field mappings in `FieldMapping[]` array (sigma field -> UDM path, with confidence from field-mappings registry)
- If a Sigma field has no UDM mapping, add it to diagnostics as a warning and include it as unmapped in fieldMappings
- Normalize rule name: replace spaces/hyphens with underscores, lowercase, strip non-alphanumeric

**2. YARA-L -> Sigma Translation (`yaralToSigma`):**

Input: YARA-L rule text string
Output: Sigma YAML string

Steps:
- Parse the YARA-L rule (regex-based, same approach as yaral-adapter's parseYaralRule -- either import if exported, or duplicate the parsing logic here)
- If parse fails, return error TranslationResult
- Detect multi-event rules: if more than one unique event variable name exists (e.g., $e1 and $e2), this is a multi-event correlation rule
- For multi-event rules: translate ONLY the first event variable's predicates, and populate `untranslatableFeatures` with:
  - `"Multi-event correlation with {N} event variables ($e1, $e2, ...) -- only $e1 predicates translated"`
  - `"Condition expression '{condition}' not preserved"`
  - If match: section exists: `"Match section (grouping/aggregation) not preserved"`
  - If outcome: section exists: `"Outcome section (risk scoring) not preserved"`

- For each event predicate in the selected variable:
  - Reverse-map UDM path to Sigma field using `getAllFieldMappings()` -- find entry where `udmPath` matches the predicate's fieldPath, return `sigmaField`
  - If no reverse mapping found, use the raw UDM path as the Sigma field name and add a diagnostic warning
  - For regex predicates: determine Sigma modifier:
    - `/^value/` -> `|startswith`
    - `/value$/` -> `|endswith`
    - `/value/` with nocase -> `|contains`
    - other regex -> `|re`
  - For exact string predicates: no modifier

- Build Sigma YAML:
```yaml
title: "{ruleName normalized}"
id: {generate UUID}
status: test
description: "Translated from YARA-L rule: {ruleName}"
author: "{meta.author || 'Detection Lab'}"
date: "{meta.created || today}"
tags:
  - {from meta.mitre_attack if present, prefix with "attack."}
logsource:
  category: "{eventTypeToLogsource(eventType)}"
  product: windows
detection:
  selection:
    {sigmaField}: {value}
    {sigmaField|modifier}: {value}
  condition: selection
falsepositives:
  - Unknown
level: "{chronicleSeverityToSigma(meta.severity)}"
```

- UDM event_type to Sigma logsource category (reverse of Sigma->YARA-L):
  - PROCESS_LAUNCH -> process_creation
  - FILE_CREATION -> file_event
  - NETWORK_CONNECTION -> network_connection
  - NETWORK_DNS -> dns_query
  - REGISTRY_MODIFICATION -> registry_set
  - USER_LOGIN -> authentication
  - default -> process_creation

- Chronicle severity to Sigma level: CRITICAL->critical, HIGH->high, MEDIUM->medium, LOW->low, INFORMATIONAL->informational

**3. TranslationProvider:**

```typescript
const yaralTranslationProvider: TranslationProvider = {
  canTranslate(from: FileType, to: FileType): boolean {
    return (from === "sigma_rule" && to === "yaral_rule") ||
           (from === "yaral_rule" && to === "sigma_rule");
  },
  async translate(request: TranslationRequest): Promise<TranslationResult> {
    if (request.sourceFileType === "sigma_rule" && request.targetFileType === "yaral_rule") {
      return sigmaToYaral(request.source);
    }
    if (request.sourceFileType === "yaral_rule" && request.targetFileType === "sigma_rule") {
      return yaralToSigma(request.source);
    }
    return { success: false, output: null, diagnostics: [{ severity: "error", message: "Unsupported translation pair" }], fieldMappings: [], untranslatableFeatures: [] };
  },
};
```
