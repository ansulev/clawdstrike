Create two files:

**gutter-types.ts** -- Shared types for gutter extensions:
- `GuardLineRange`: `{ guardId: string; fromLine: number; toLine: number }` -- identifies which lines in the YAML correspond to a guard config section.
- `CoverageGap`: `{ guardId: string; line: number; uncoveredTechniques: string[] }` -- a guard at a given line with uncovered MITRE techniques.
- `parseGuardRanges(doc: Text): GuardLineRange[]` -- parses a YAML document to find guard config sections. Implementation: scan lines for patterns matching guard IDs from `ALL_GUARD_IDS` as YAML keys (e.g., `  forbidden_path:` at indentation level 2 or 4 under a `guards:` key). Each range starts at the guard key line and extends until the next sibling key at the same indentation or end of the guards block. Use `@codemirror/state`'s `Text` type.
- `computeCoverageGaps(guardRanges: GuardLineRange[], yamlContent: string): CoverageGap[]` -- for each guard range, checks if the guard is enabled (look for `enabled: true` within the range), maps via `GUARD_TECHNIQUE_MAP` to MITRE technique IDs, then uses `extractPolicyTechniques(yamlContent)` to find all covered techniques. Guards whose techniques are all covered have no gap. Guards with uncovered techniques produce a CoverageGap entry.

**guard-gutter.ts** -- CodeMirror gutter extension for "Run Test" buttons:
- `updateGuardRanges`: a `StateEffect<GuardLineRange[]>` for pushing parsed guard ranges into editor state.
- `guardRangesField`: a `StateField<GuardLineRange[]>` that stores current guard ranges, reduced by the effect.
- `RunTestMarker extends GutterMarker`: renders a play button (triangle SVG or Unicode) in the gutter. The marker stores the `guardId`. Its `toDOM()` method creates a `<span>` with:
  - class `cm-guard-test-btn`
  - a play icon (use `\u25B6` or a tiny inline SVG triangle)
  - `title="Run test: {guardName}"` (look up name from `getGuardMeta`)
  - click handler that dispatches a `CustomEvent("clawdstrike:run-guard-test", { detail: { guardId } })` on the editor's DOM element. This allows React to listen and handle the test execution.
  - Styled: visibility hidden by default, visible on `.cm-gutterElement:hover .cm-guard-test-btn` (handled via theme).
- `guardTestGutter(onRunTest: (guardId: string) => void)`: returns `Extension[]` containing:
  1. The StateField
  2. `gutter({ class: "cm-guard-test-gutter", markers: ... })` that reads `guardRangesField` and places a `RunTestMarker` on the first line of each guard range.
  3. Theme styling: `.cm-guard-test-gutter .cm-gutterElement` width 20px, `.cm-guard-test-btn` opacity 0 by default, `.cm-gutterElement:hover .cm-guard-test-btn` opacity 1 with transition, icon color `#d4a84b` (gold), hover color `#3dbf84` (green).
  4. An `EditorView.domEventHandlers({ click })` that intercepts clicks on `.cm-guard-test-btn` elements and calls `onRunTest(guardId)`.

Use the brand colors from the existing theme: gold `#d4a84b`, green `#3dbf84`, steel `#6f7f9a`, bg `#0b0d13`.
