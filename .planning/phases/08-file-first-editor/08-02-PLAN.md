2. **Add dirty dot indicator** inside the PaneTab button, positioned BEFORE the label span. When `view.dirty` is true AND the tab is active, show a filled gold dot. When `view.dirty` is true AND the tab is NOT active, show the dot dimmed. When not dirty, show nothing:
   ```tsx
   {view.dirty && (
     <span
       className={cn(
         "h-2 w-2 shrink-0 rounded-full",
         isActive ? "bg-[#d4a84b]" : "bg-[#d4a84b]/50",
       )}
       aria-label="Unsaved changes"
     />
   )}
   ```
   Place this immediately after the opening `<button>` content start, before the label `<span>`.

3. **Add file-type color dot** -- a small colored circle showing the file type. Only render when `view.fileType` is present AND the route starts with "/file/" (so non-file tabs like Home, Guards, etc. don't get dots):
   ```tsx
   {view.fileType && view.route.startsWith("/file/") && (
     <span
       className="h-1.5 w-1.5 shrink-0 rounded-full"
       style={{ backgroundColor: FILE_TYPE_COLORS[view.fileType] ?? "#6f7f9a" }}
       aria-label={`File type: ${view.fileType}`}
     />
   )}
   ```
   Place this after the dirty dot, before the label.

4. **Modify the close button behavior** for dirty tabs: When `view.dirty` is true, the close button should show a filled dot instead of the X icon when the tab is NOT hovered (and revert to X on hover). This matches VS Code behavior where dirty tabs show a dot that becomes X on hover:
   - When `view.dirty` is true AND NOT being hovered by the group, show the dirty dot in place of X.
   - The existing `group-hover/tab:opacity-100` class already handles hover visibility of the X button.
   - Simpler approach: when `view.dirty` AND the close button is NOT being hovered, show the dot. Use a local state or the existing group hover:
   ```tsx
   <span
     role="button"
     tabIndex={-1}
     aria-label={`Close ${view.label}`}
     className={cn(
       "flex h-5 w-5 shrink-0 items-center justify-center rounded-sm transition-colors",
       isActive
         ? "opacity-100 text-[#6f7f9a]/40 hover:bg-[#2a1115] hover:text-[#ffb8b8]"
         : "opacity-0 group-hover/tab:opacity-100 text-[#6f7f9a]/40 hover:bg-[#2a1115] hover:text-[#ffb8b8]",
     )}
     onClick={(e) => {
       e.stopPropagation();
       usePaneStore.getState().closeView(paneId, view.id);
     }}
   >
     {view.dirty && isActive ? (
       <span className="h-2 w-2 rounded-full bg-[#d4a84b] group-hover/tab:hidden" />
     ) : null}
     <IconX size={14} stroke={1.8} className={view.dirty && isActive ? "hidden group-hover/tab:block" : ""} />
   </span>
   ```
   NOTE: Keep this simple. If the CSS toggle between dot/X is complex, just always show X -- the dedicated dirty dot before the label already communicates the state. The close-button dot is a nice-to-have, not critical.
