    **Root cause (App.test.tsx):** The test renders the full `<App />` tree which reaches `ResizablePanelGroup` (from `@/components/ui/resizable`).     The test also does not mock `ActivityBar` (imported from `@/features/activity-bar/components/activity-bar`), `SidebarPanel`, `SidebarResizeHandle`, or the many hooks in `WorkbenchBootstraps` (`useOperator`, `useFleetConnection`, `useSignalCorrelator`, `useHintSettingsSafe`). These pull in deep dependency chains (NATS, SSE, operator store) that break in jsdom.

    **Fix setup.ts:** Add a minimal `ResizeObserver` polyfill to the global test setup so ALL tests in the suite can use components that depend on it:

    ```typescript
    // ResizeObserver polyfill for jsdom
    if (typeof globalThis.ResizeObserver === "undefined") {
      globalThis.ResizeObserver = class ResizeObserver {
        observe() {}
        unobserve() {}
        disconnect() {}
      } as unknown as typeof globalThis.ResizeObserver;
    }
    ```

    Place this BEFORE the `beforeEach` block in setup.ts. This is a static polyfill -- it never fires callbacks, but prevents "ResizeObserver is not defined" crashes.
