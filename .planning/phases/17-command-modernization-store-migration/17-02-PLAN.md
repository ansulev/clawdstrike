useWorkbench() consumers fall into 3 patterns:

PATTERN A — Read-only state (most common, ~20 files):
  Bridge: `const { state } = useWorkbench()`
  Uses: `state.activePolicy`, `state.yaml`, `state.dirty`, `state.validation`, `state.nativeValidation`, `state.savedPolicies`, `state.filePath`, `state.ui.sidebarCollapsed`, `state.ui.activeEditorTab`
  Direct replacement:
  ```typescript
  const activeTabId = usePolicyTabsStore(s => s.activeTabId);
  const activeTab = usePolicyTabsStore(s => s.tabs.find(t => t.id === s.activeTabId));
  const editState = usePolicyEditStore(s => s.editStates.get(activeTabId));
  // Then: editState?.policy for state.activePolicy
  //       editState?.yaml for state.yaml
  //       activeTab?.dirty for state.dirty
  //       activeTab?.filePath for state.filePath
  //       editState?.validation for state.validation
  //       editState?.nativeValidation for state.nativeValidation
  // For savedPolicies:
  const savedPolicies = usePolicyTabsStore(s => s.savedPolicies);
  // For UI state:
  const sidebarCollapsed = useWorkbenchUIStore(s => s.sidebarCollapsed);
  ```

PATTERN B — State + dispatch (10 files):
  Bridge: `const { state, dispatch } = useWorkbench()`
  dispatch maps actions to store methods. Replace dispatch calls:
  - `dispatch({ type: "SET_POLICY", policy })` -> `usePolicyEditStore.getState().updatePolicy(activeTabId, policy, activeTab.fileType)`
  - `dispatch({ type: "SET_YAML", yaml })` -> `usePolicyEditStore.getState().setYaml(activeTabId, yaml, ...)`
  - `dispatch({ type: "UPDATE_GUARD", guardId, config })` -> `usePolicyEditStore.getState().updateGuard(activeTabId, guardId, config, activeTab.fileType)`
  - `dispatch({ type: "TOGGLE_GUARD", guardId, enabled })` -> `usePolicyEditStore.getState().toggleGuard(activeTabId, guardId, enabled, activeTab.fileType)`
  - `dispatch({ type: "UPDATE_SETTINGS", settings })` -> `usePolicyEditStore.getState().updateSettings(activeTabId, settings, activeTab.fileType)`
  - `dispatch({ type: "UPDATE_META", ...meta })` -> `usePolicyEditStore.getState().updateMeta(activeTabId, meta, activeTab.fileType)`
  - `dispatch({ type: "UPDATE_ORIGINS", origins })` -> `usePolicyEditStore.getState().updateOrigins(activeTabId, origins, activeTab.fileType)`
  - `dispatch({ type: "SET_SIDEBAR_COLLAPSED", collapsed })` -> `useWorkbenchUIStore.getState().setSidebarCollapsed(collapsed)`
  - `dispatch({ type: "SET_EDITOR_TAB", tab })` -> `useWorkbenchUIStore.getState().setActiveEditorTab(tab)`
  - `dispatch({ type: "MARK_CLEAN" })` -> `usePolicyEditStore.getState().markClean(activeTabId)`
  - `dispatch({ type: "SET_NATIVE_VALIDATION", payload })` -> `usePolicyEditStore.getState().setNativeValidation(activeTabId, payload)`

PATTERN C — State + helper methods (5 files: workbench-topbar, import-export, catalog-browser, library-gallery, policy-card):
  Bridge: `const { state, loadPolicy, openFile, openFileByPath, exportYaml, saveCurrentPolicy } = useWorkbench()`
  For `loadPolicy(policy)`: If current tab is not a policy type, call `usePolicyTabsStore.getState().newTab({ policy })`, else call `usePolicyEditStore.getState().updatePolicy(activeTabId, policy, activeTab.fileType)`.
  For `openFile()`: Call `openDetectionFile()` from tauri-bridge, then `usePolicyTabsStore.getState().openTabOrSwitch(result.path, result.fileType, result.content)` + `pushRecentFile(result.path)`.
  For `openFileByPath(path)`: Call `readDetectionFileByPath(path)` then `usePolicyTabsStore.getState().openTabOrSwitch(...)` + `pushRecentFile(...)`.
  For `exportYaml()`: Read yaml from `usePolicyEditStore.getState().getTabEditState(activeTabId)?.yaml`, create Blob + download link.
  For `saveCurrentPolicy()`: Read policy from editState, create SavedPolicy, call `usePolicyTabsStore.getState().savePolicyToLibrary(savedPolicy)`.

useMultiPolicy() consumers fall into 2 patterns:

PATTERN D — Tabs + dispatch (policy-tab-bar, bulk-operations-dialog, policy-command-center, sentinel-swarm-pages, hunt-layout, use-auto-save):
  Bridge: `const { tabs, multiDispatch, multiState, canAddTab } = useMultiPolicy()`
  Direct:
  ```typescript
  const tabs = usePolicyTabsStore(s => s.tabs);
  const canAddTab = usePolicyTabsStore(s => s.tabs.length < 25);
  ```
  Replace multiDispatch calls with direct store methods (see PATTERN B dispatch mapping above, plus tab-lifecycle actions on usePolicyTabsStore).

PATTERN E — Active tab read (deploy-panel, sdk-integration-tab, status-bar, simulator-layout, test-runner-panel, home-page, desktop-layout):
  Bridge: `const { activeTab } = useMultiPolicy()` or `const { tabs } = useMultiPolicy()`
  activeTab was a full PolicyTab. Direct:
  ```typescript
  const activeTab = usePolicyTabsStore(s => s.tabs.find(t => t.id === s.activeTabId));
  // If component reads activeTab.yaml/policy/validation, also:
  const editState = usePolicyEditStore(s => s.editStates.get(usePolicyTabsStore.getState().activeTabId));
  ```

TabMeta vs PolicyTab fields mapping:
  TabMeta has: id, documentId, name, filePath, dirty, fileType
  PolicyTab additionally had: policy, yaml, validation, nativeValidation, testSuiteYaml, _undoPast, _undoFuture, _cleanSnapshot
  The extra fields come from TabEditState in policy-edit-store.
