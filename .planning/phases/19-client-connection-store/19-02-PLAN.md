Create `apps/workbench/src/features/presence/stores/presence-store.ts` following the exact Zustand + immer + createSelectors pattern used by activity-bar-store and fleet-connection-store.

**Store state interface:**
```typescript
export interface PresenceStoreState {
  /** Current WebSocket connection state */
  connectionState: PresenceConnectionState;
  /** Connection error message, if any */
  connectionError: string | null;
  /** This operator's assigned analyst ID (set on welcome) */
  localAnalystId: string | null;
  /** This operator's assigned color (set on welcome) */
  localColor: string | null;
  /** Connected analysts keyed by fingerprint */
  analysts: Map<string, AnalystPresence>;
  /** Reverse index: file path -> Set of fingerprints viewing that file */
  viewersByFile: Map<string, Set<string>>;

  actions: PresenceActions;
}
```

**PresenceActions interface:**
```typescript
interface PresenceActions {
  /** Process a raw server message and update store state accordingly */
  handleServerMessage: (msg: ServerMessageRaw) => void;
  /** Update connection state (called by PresenceSocket) */
  setConnectionState: (state: PresenceConnectionState) => void;
  /** Reset store to empty offline defaults */
  reset: () => void;
}
```

**handleServerMessage implementation** (the core message router — uses immer mutations):

Switch on `msg.type`:

- `"welcome"`:
  - Set `localAnalystId` to `msg.analyst_id`
  - Set `localColor` to `msg.color`
  - Clear existing analysts Map
  - Clear existing viewersByFile Map
  - For each analyst in `msg.roster`, call `parseAnalystInfo(analyst)` and insert into analysts Map keyed by fingerprint
  - Rebuild viewersByFile from roster (any analyst with active_file gets added to that file's Set)

- `"analyst_joined"`:
  - Call `parseAnalystInfo(msg.analyst)` and insert into analysts Map
  - If analyst has active_file, add fingerprint to viewersByFile for that file

- `"analyst_left"`:
  - Get the analyst from Map before deleting (to clean up viewersByFile)
  - If analyst had activeFile, remove from viewersByFile for that file
  - Delete from analysts Map
  - Clean up empty Sets in viewersByFile

- `"analyst_viewing"`:
  - Update analyst's activeFile in Map
  - Remove fingerprint from ALL viewersByFile entries (may have been viewing another file)
  - Add fingerprint to viewersByFile for `msg.file_path`
  - Clean up empty Sets

- `"analyst_left_file"`:
  - Update analyst's activeFile to null in Map
  - Remove fingerprint from viewersByFile for `msg.file_path`
  - Clean up empty Set if needed

- `"analyst_cursor"`:
  - Update analyst's cursor to `{ line: msg.line, ch: msg.ch }` in Map
  - Update analyst's lastSeen to Date.now()

- `"analyst_selection"`:
  - Update analyst's selection to `{ anchorLine: msg.anchor_line, anchorCh: msg.anchor_ch, headLine: msg.head_line, headCh: msg.head_ch }` in Map
  - Update analyst's lastSeen to Date.now()

- `"heartbeat_ack"`: No-op (or optionally update a lastHeartbeatAck timestamp)

- `"error"`:
  - Set connectionError to `msg.message`
  - Console warn the error

**CRITICAL immer gotcha with Map/Set:** Zustand immer middleware does NOT auto-draft Maps and Sets by default. Use `enableMapSet()` from immer at the top of the file:
```typescript
import { enableMapSet } from "immer";
enableMapSet();
```
This MUST be called before any store creation. Without this, immer mutations on Map/Set will silently produce broken state.

**reset() action:**
- Set connectionState to "idle"
- Set connectionError to null
- Set localAnalystId to null
- Set localColor to null
- Clear analysts Map (new Map())
- Clear viewersByFile Map (new Map())

**Default state (offline degradation):**
- connectionState: "idle"
- connectionError: null
- localAnalystId: null
- localColor: null
- analysts: new Map()
- viewersByFile: new Map()

The default state means the workbench is fully functional without hushd — all presence-dependent UI just shows empty/hidden state.

Create `apps/workbench/src/features/presence/use-presence-connection.ts` — a React hook that creates and manages the PresenceSocket lifecycle, wiring it to the presence-store.

**Module-level singleton:** Keep a single PresenceSocket instance at module scope (like how use-fleet-connection.ts keeps `fleetEventStream` at module scope). This ensures only one WebSocket connection regardless of how many components use the hook.

```typescript
import { useEffect, useRef } from "react";
import { PresenceSocket } from "./presence-socket";
import { usePresenceStore } from "./stores/presence-store";
import { useFleetConnectionStore } from "@/features/fleet/use-fleet-connection";
import { useOperatorStore } from "@/features/operator/stores/operator-store";
import type { PresenceConnectionState, ServerMessageRaw } from "./types";

let presenceSocket: PresenceSocket | null = null;

export function usePresenceConnection(): void {
  // ...
}
```

**Hook implementation:**

1. Read fleet connection state: `const hushdUrl = useFleetConnectionStore((s) => s.connection.hushdUrl)`
2. Read fleet connected status: `const fleetConnected = useFleetConnectionStore((s) => s.connection.connected)`
3. Read operator identity: `const operator = useOperatorStore((s) => s.currentOperator)`

4. **useEffect** with deps `[hushdUrl, fleetConnected, operator?.fingerprint]`:
   - If NOT fleetConnected OR no hushdUrl OR no operator: disconnect existing socket, reset store, return cleanup
   - If presenceSocket already exists with same hushdUrl: do nothing (already connected)
   - Otherwise: disconnect old socket if exists, then create new PresenceSocket:

```typescript
presenceSocket = new PresenceSocket({
  hushdUrl,
  getApiKey: () => useFleetConnectionStore.getState().actions.getCredentials().apiKey,
  getIdentity: () => {
    const op = useOperatorStore.getState().currentOperator;
    if (!op) return null;
    return { fingerprint: op.fingerprint, displayName: op.displayName, sigil: op.sigil };
  },
  onMessage: (msg: ServerMessageRaw) => {
    usePresenceStore.getState().actions.handleServerMessage(msg);
  },
  onStateChange: (state: PresenceConnectionState) => {
    usePresenceStore.getState().actions.setConnectionState(state);
  },
  onReconnect: () => {
    // On reconnect, the server will send a fresh Welcome with full roster
    // No additional action needed — handleServerMessage("welcome") resets state
  },
});
presenceSocket.connect();
```

5. **Cleanup function:** On unmount, disconnect socket, reset store, set presenceSocket to null.

**CRITICAL design decisions:**
- `getApiKey` is a FUNCTION that fetches fresh credentials on every connect/reconnect. It calls `useFleetConnectionStore.getState()` — NOT a captured value. This prevents stale credential bugs (Pitfall 2 from research).
- `getIdentity` is also a function that reads current operator state at call time.
- The hook is called from a single high-level component (e.g., App.tsx or a shell component). It does NOT render anything.
- When fleet disconnects, the presence socket also disconnects gracefully.

**Also export a helper for sending messages from outside React** (used by Phase 21 CM6 extension):
```typescript
export function getPresenceSocket(): PresenceSocket | null {
  return presenceSocket;
}
```

This allows the CM6 ViewPlugin in Phase 21 to call `getPresenceSocket()?.send({ type: "cursor", ... })` without going through React.
