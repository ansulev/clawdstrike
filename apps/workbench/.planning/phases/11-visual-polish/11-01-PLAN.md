    **Root cause of POLISH-01:** The pane store creates an initial root with a `/home` view. Then `DesktopLayout.useLayoutEffect` calls `syncRoute(currentRoute)`. The `syncRoute` function calls `setPaneActiveRoute` which REPLACES the active view's route — it does not add a new tab. However, in Tauri's HashRouter, the mount cycle can cause `syncRoute` to be called twice in quick succession: once during initial render (route `/` normalizes to `/home`), and again after `navigate("/home", { replace: true })` triggers a re-render. Under certain timing conditions (React strict mode, Suspense re-mounts), this can result in `syncRoute` being called while the active view has already been switched by a concurrent effect, causing `setPaneActiveRoute` to overwrite a non-Home view or create an inconsistency.

    **Fix syncRoute in pane-store.ts (line 80-90):** Before calling `setPaneActiveRoute`, check if the normalized route already matches the active view's route (early return — no-op). Also check if the route exists as ANY view in the active pane group; if so, switch to that view instead of replacing the current active view.

    Replace the `syncRoute` implementation:
    ```typescript
    syncRoute: (route) => {
      const normalized = normalizeWorkbenchRoute(route);
      const state = get();
      const activePane = getActivePane(state.root, state.activePaneId)
        ?? getFirstPaneGroup(state.root);

      // If the active view already shows this route, no-op.
      const activeView = getPaneActiveView(activePane);
      if (activeView && normalizeWorkbenchRoute(activeView.route) === normalized) {
        return;
      }

      // If another tab in the active pane already has this route, focus it.
      const existingView = activePane.views.find(
        (v) => normalizeWorkbenchRoute(v.route) === normalized,
      );
      if (existingView) {
        set((s) => ({
          root: setActivePaneView(s.root, s.activePaneId, existingView.id),
        }));
        return;
      }

      // Otherwise, replace the active view's route (original behavior).
      set((s) => ({
        root: setPaneActiveRoute(
          s.root,
          s.activePaneId,
          normalized,
          getWorkbenchRouteLabel(normalized),
        ),
      }));
    },
    ```
