# Docs

Documentation domains and intent.

Layout:

1. `docs/src/` - public mdBook user/contributor docs.
2. `docs/plans/` - implementation plans and architecture decisions.
3. `docs/specs/` - normative specification documents.
4. `docs/research/` - exploratory, non-normative analysis.
5. `docs/roadmaps/` - time-based planning artifacts.
6. `docs/audits/` - point-in-time quality and architecture audits.

Canonical source and precedence rules are defined in `docs/DOCS_MAP.md`.
